Мобильное приложение · спецификация

Спецификация мобильного приложения

Автор: Кагиров А.-Х.А. · Свидетельство о депонировании №4011265 от 19.12.2024 · Center Group Company · Engagement: CPFSR-RF-DESIGN-2026

Спецификация мобильного приложения ЦП РСФСР

Платформы: iOS / Android / Windows. Целевая аудитория: залогодатели — физические и юридические лица, представители государственных предприятий, ИП, НКО и фондов. Ключевые цели: обеспечить безопасное удалённое управление активами, выпуск и обслуживание ЦФА, получение вознаграждения.

1. Технологический стек

1.1. Кросс-платформенная реализация

  • Базовая платформа: PWA (HTML/CSS/JS + TypeScript), Service Worker для offline-режима и фонового обновления, манифест с shortcuts и share_target.
  • iOS / Android wrapper: Capacitor v6+ (нативные iOS/Android-обёртки над PWA-ядром); единый кодовая база, нативный доступ к биометрии, ГОСТ-СКЗИ, Secure Enclave/Strongbox, certificate pinning, push, share_target.
  • Windows desktop: PWA через Edge WebView2 (отдельная инсталляция через Microsoft Store / MSIX) либо edge_side_panel.
  • Общая бизнес-логика: TypeScript + WebAssembly-модули для критичных функций (криптография, подпись).
  • Дизайн-система: на основе бренд-токенов проекта (_shared/brand.css), с native-overrides для iOS (SF Pro) и Android (Roboto).

1.2. Архитектура клиентского приложения

  • Pattern: PWA Shell + lazy-loaded screens.
  • State management: внутренний state-store в app.js (vanilla JS) либо адаптер на Zustand при росте сложности.
  • Навигация: PWA back-stack через History API + drawer-aside в полноразмерных режимах.
  • API-клиент: автогенерация из OpenAPI 3.1.

1.3. Хранение данных на устройстве

  • Sensitive data: iOS Keychain / Android Keystore / Windows Credential Locker — только зашифрованные токены.
  • Биометрические данные не хранятся локально — только идентификаторы для верификации через ЕБС.
  • Локальный кеш — SQLCipher (шифрованный SQLite).

1.4. Криптография

  • УКЭП в мобильном устройстве — через КриптоПро ЭП (мобильный SDK).
  • Хеширование — ГОСТ Р 34.11-2012.
  • TLS-каналы — с использованием ГОСТ-сертификатов.

2. Безопасность

2.1. Идентификация и аутентификация

  1. Первичная авторизация — через ЕСИА (OpenID Connect) с подтверждённой учётной записью.
  2. Дополнительные факторы:
    • Биометрия устройства (TouchID / FaceID / Windows Hello) для доступа к приложению.
    • Биометрия через ЕБС (Положение Банка России №482-П) для подписания критичных операций.
    • OTP (SMS / push) для критичных операций (подписание смарт-контракта).

2.2. Защита приложения

  • Anti-tampering: Mobile Application Shielding (MAS) — отечественные решения класса.
  • Jailbreak / Root Detection — отказ работы на скомпрометированных устройствах.
  • Anti-debug — обфускация, защита от runtime-инъекций.
  • Anti-screenshot для критичных экранов (смарт-контракт, OTP).
  • Certificate pinning на TLS-каналах.

2.3. Защита от фрода

  • Поведенческая аналитика (поведенческая биометрия).
  • Device fingerprinting.
  • Геолокация и временные паттерны.

2.4. Хранение секретов

  • Только в TEE (Trusted Execution Environment) / Secure Enclave.
  • Никаких ключей в общей памяти приложения.

2.5. Соответствие

  • 152-ФЗ (защита ПДн).
  • 187-ФЗ (КИИ) — приложение является частью значимого объекта КИИ 1 категории.
  • Указание Банка России №6635-У (инциденты ИБ).
  • Стандарт OWASP MASVS Level 2+.
  • Соответствие WCAG 2.2 AA (доступность).

3. Функциональные модули

3.1. Регистрация и идентификация

  • Авторизация через ЕСИА.
  • Верификация биометрии через ЕБС.
  • Привязка устройства.
  • Согласия на обработку ПДн.

3.2. Личный кабинет

  • Дашборд: общий баланс активов в работе, текущий доход, портфель ЦФА.
  • Список активов с фильтрами и поиском.
  • Каждый актив: характеристики, история, текущий ЦФА.

3.3. Подача актива на залог

  • Выбор типа имущества.
  • Подгрузка данных из государственных реестров (ЕГРН, Роспатент, депозитарии).
  • Прикрепление документов.
  • Заявка на оценку.

3.4. Оценка

  • Просмотр статуса заявки.
  • Результат оценки с возможностью обжалования.
  • Информация о страховании.

3.5. Смарт-контракт

  • Конструктор условий.
  • Просмотр полного текста контракта.
  • Электронное подписание УКЭП (мобильный сертификат).

3.6. Управление ЦФА

  • Список активных ЦФА.
  • Состояние каждого ЦФА в DLT-реестре.
  • Ожидаемые события (мена, страховые случаи, окончание срока).

3.7. Доходы

  • История выплат.
  • Сводка по кварталам и годам.
  • Скачивание справки 2-НДФЛ.

3.8. Уведомления

  • Push-уведомления по критичным событиям.
  • Уведомления через Госуслуги (для согласий, изменений).

3.9. Профиль

  • Управление контактными данными.
  • Управление безопасностью (биометрия, ОТП-канал).
  • Управление согласиями.
  • Реквизиты для выплат.

3.10. Помощь и поддержка

  • FAQ.
  • Чат-бот.
  • Связь с оператором поддержки.

5. Релизный план

5.1. iOS / Android

  • Бета-релиз через TestFlight / Google Play Internal Testing — месяц 10.
  • Закрытое тестирование — месяц 12.
  • Открытый запуск — месяц 18 (одновременно с платформой).
  • Размещение в RuStore (Минцифры).

5.2. Windows / Desktop (Capacitor v6+ для desktop / PWA installable)

  • Бета-релиз — месяц 14 (PWA через Edge / Chrome «Install app»; полноценный Capacitor Windows bundle — месяц 16).
  • Открытый запуск — месяц 18.
  • Размещение в Microsoft Store (через MSIX-пакет, генерируемый Capacitor) + альтернативные каналы (web-distribution PWA).
  • Преимущество единой PWA-кодобазы: те же исходники, что для iOS/Android — нет параллельного .NET MAUI стека.

5.3. Обновления

  • Auto-update через стандартные сторы.
  • Принудительные обновления для критичных уязвимостей через in-app механизм.

6. Тестирование

6.1. Виды тестирования

  • Unit-тесты (Vitest + @testing-library/dom для PWA-логики, JUnit/XCTest для нативных Capacitor-плагинов) — покрытие ≥ 80%.
  • Integration-тесты (Playwright Component Testing) — критичные сценарии.
  • E2E-тесты (Playwright cross-browser + Capacitor + Appium для нативных wrappers) — основные пользовательские флоу для iOS / Android / desktop.
  • Performance-тесты (Lighthouse CI ≥ 90 по всем 4 метрикам PWA) на разных устройствах.
  • Security-тесты (OWASP MASVS L2+ FT-021..025, ZAP / Burp, OWASP Mobile Top 10, FSTEC-сертифицированные сканеры).
  • Accessibility-тесты (axe-core, WCAG 2.2 AA, screen reader iOS VoiceOver / Android TalkBack).

6.2. Парк устройств

  • iOS: iPhone 13 / 14 / 15 / 16; iPad 9-gen / Pro.
  • Android: Samsung Galaxy S22+, Xiaomi Redmi Note 12, Realme C55 (бюджетный сегмент).
  • Windows: Surface Pro 9, типовые рабочие станции.

6.3. CI/CD

  • Сборка в защищённой среде с подписью УКЭП.
  • Автоматическое тестирование на каждом merge.
  • Релизные ветки + теги.

7. Соответствие магазинам приложений

7.1. RuStore (приоритет)

  • Полное соответствие требованиям RuStore.
  • Подача в магазин в первые 90 дней после публичного запуска.

7.2. App Store

  • Соответствие правилам Apple.
  • Возможные ограничения по криптовалютам — ЦФА не являются криптовалютой; правила Apple про токенизированные финансовые активы — допускают, при наличии лицензии.

7.3. Google Play

  • Соответствие политике Play.
  • Подача в Финансовые приложения с подтверждением регистрации в РФ.

7.4. Microsoft Store

  • Соответствие политике Store.
  • Категория Finance / Productivity.

8. Локализация и доступность

8.1. Локализация

  • Основной язык: русский.
  • Полная поддержка кириллицы во всех экранах.
  • Локализация дат, чисел, валют.
  • Английский язык — для интернациональной аудитории (опционально).

8.2. Доступность

  • WCAG 2.2 AA.
  • Голосовое управление через VoiceOver / TalkBack / Narrator.
  • Динамическое увеличение шрифтов.
  • Высокая контрастность.
