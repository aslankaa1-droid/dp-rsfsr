# Описание программы

## ЦП РСФСР — мобильное приложение

**Условное обозначение:** РСФСР.МП.02.ОП
**Стандарт:** ГОСТ 19.402-78
**Правообладатель:** Кагиров Абдул-Хаким Ахмадович

---

## 1. Общие сведения

### 1.1. Обозначение и наименование программы
Полное наименование: «ЦП РСФСР — мобильное приложение залогодателя».
Условное: РСФСР.МП.

### 1.2. Программное обеспечение, необходимое для функционирования
Программа функционирует автономно. Необходимое окружение:
- операционная система: iOS 15+, Android 9+, Windows 10+;
- веб-браузер для PWA-режима с поддержкой Service Worker;
- сетевое подключение для интерактивных операций (просмотр кешированных данных доступен оффлайн).

### 1.3. Языки программирования
- HTML5 / CSS3 / JavaScript (ES2020+) — основной код PWA.
- TypeScript — серверная часть API (вне приложения).
- Capacitor v6+ (TypeScript) — обёртка для нативной упаковки (единая кодобаза для iOS / Android / Windows desktop).
- Native bridges (через Capacitor):
  - Kotlin / Java — Android-плагины (для расширений за пределами Web API);
  - Swift / Objective-C — iOS-плагины (Secure Enclave, биометрия);
  - C# / Windows App SDK — только для нативных Windows-плагинов внутри Capacitor MSIX-пакета (Toast, FileSystem ACL, KeyVault).

## 2. Функциональное назначение

### 2.1. Классы решаемых задач
Программа относится к классу клиентских мобильных приложений финансово-учётного назначения с интеграцией с государственными информационными системами.

### 2.2. Сведения о функциональных ограничениях
- Идентификация и подписание операций требуют подтверждённой учётной записи Госуслуг и доступа к ЕБС.
- Полноценное использование требует устройства с защищённым хранилищем (Secure Enclave / Trusted Execution Environment).
- Биометрия может быть недоступна на устройствах без соответствующих датчиков.
- Установка из RuStore, App Store, Google Play, Microsoft Store, либо PWA-установка через браузер.

## 3. Описание логической структуры

### 3.1. Архитектура программы
Программа построена по архитектурному принципу **Single Page Application** с использованием паттерна **Clean Architecture** и разделением на слои:

1. **Presentation Layer** — экраны и компоненты пользовательского интерфейса (HTML/CSS).
2. **Application Layer** — логика взаимодействия экранов, маршрутизация, управление состоянием (JavaScript).
3. **Domain Layer** — бизнес-сущности (Asset, CFA, SmartContract, User, Transaction, Notification).
4. **Data Layer** — взаимодействие с локальным хранилищем (LocalStorage, IndexedDB, SQLite) и сетевыми API.
5. **Infrastructure Layer** — Service Worker, push-уведомления, биометрия, криптография.

### 3.2. Структура программы
Программа состоит из:
- единого HTML-файла с встроенным CSS и JavaScript кодом (`index.html`);
- манифеста PWA (`manifest.json`);
- сервисного работника для оффлайн-режима (`sw.js`);
- иконок различных размеров (`icons/`);
- разделяемых ресурсов с веб-проектом платформы (`../../_shared/`).

### 3.3. Связи с другими программами
- **ЦП РСФСР — Backend (API)** — REST-взаимодействие через защищённый канал.
- **Госуслуги (ЕСИА)** — OAuth 2.1 / OpenID Connect.
- **Единая биометрическая система** — REST по Положению ЦБ №482-П.
- **Платёжная инфраструктура НСПК (Мир)** — для зачисления выплат.
- **СМЭВ 3** — через серверную часть.

## 4. Используемые технические средства

### Серверная часть (вне приложения)
Описана в документе «Концепция архитектуры» проекта ЦП РСФСР (`01_Техническая_документация/01_Концепция_архитектуры.md`).

### Клиентские устройства
- iPhone, iPad с поддержкой iOS 15 и выше.
- Android-устройства уровня среднего класса и выше (от 3 ГБ оперативной памяти).
- Настольные и планшетные устройства под управлением Windows 10/11.
- Любые устройства с современным веб-браузером для PWA-режима.

## 5. Вызов и загрузка

### 5.1. Установка
Подробное описание процедур установки приведено в Руководстве по установке и администрированию (РСФСР.МП.05).

Базовые пути установки:
- **RuStore (Android)** — основной канал для российских пользователей.
- **App Store (iOS)** — при наличии регуляторных одобрений.
- **Google Play (Android)** — при наличии регуляторных одобрений.
- **Microsoft Store (Windows)** — при наличии регуляторных одобрений.
- **PWA** — установка из браузера через диалог «Добавить на главный экран».

### 5.2. Запуск
Запуск осуществляется через стандартный механизм операционной системы (нажатие на иконку приложения).

### 5.3. Завершение
Завершение работы происходит при сворачивании приложения. Сессия пользователя сохраняется в защищённом хранилище и при следующем запуске восстанавливается с требованием биометрической или PIN-аутентификации.

## 6. Входные данные

### 6.1. Источники
- Действия пользователя (тапы, ввод текста).
- Данные из ЕСИА при авторизации.
- Биометрические подтверждения через ЕБС.
- Push-уведомления от серверной части.
- Данные из ЕГРН, Роспатента, депозитарных систем (через серверную часть).

### 6.2. Форматы
- JSON для всех сетевых обменов.
- Стандартные форматы платежей и нотификаций по СМЭВ 3.
- Биометрические подтверждения по форматам ЕБС.

## 7. Выходные данные

### 7.1. Адресаты
- Серверная часть ЦП РСФСР (подача актива, выпуск ЦФА, подписания, запросы выплат).
- Госуслуги (для получения официальных уведомлений).
- Пользователь (визуализация на экране).

### 7.2. Форматы
- JSON для API-запросов.
- ПДФ-выгрузки документов (договоры, справки).
- Excel/CSV — экспорт истории и доходов.

## 8. Контрольный пример

Полный сценарий проверки приведён в документе «Программа и методика испытаний» (РСФСР.МП.10).

Сокращённый базовый сценарий:
1. Запуск приложения.
2. Авторизация через ЕСИА (имитация в режиме демонстрации).
3. Подача актива на залог.
4. Получение результата оценки.
5. Настройка и подписание смарт-контракта.
6. Проверка выпуска ЦФА.
7. Проверка отображения выпущенного ЦФА в портфеле.
8. Проверка получения уведомления о выпуске.
