# Описание применения

## ЦП РСФСР — мобильное приложение

**Условное обозначение:** РСФСР.МП.03.ОПР
**Стандарт:** ГОСТ 19.502-78

---

## 1. Назначение программы

### 1.1. Возможности
Приложение реализует клиентский функционал Цифровой Платформы РСФСР для собственников активов:
- регистрация и идентификация;
- подача имущества на залог;
- выпуск и обслуживание цифровых финансовых активов (ЦФА);
- получение дохода от размещения;
- управление портфелем активов и ЦФА.

### 1.2. Основные характеристики
- 23 пользовательских экрана.
- 4 темы оформления (светлая, тёмная, мягкая, контрастная A11Y).
- 4 языка интерфейса (русский, английский, французский, арабский с RTL).
- Полная WCAG 2.2 AA доступность.
- Оффлайн-режим через Service Worker.
- Поддержка установки как нативного приложения на iOS, Android, Windows.
- Поддержка PWA-режима для любого современного браузера.

### 1.3. Сведения об ограничениях
- Полноценные операции (выпуск ЦФА, подписание) требуют сетевого подключения.
- Подтверждение операций — через ЕСИА и ЕБС.
- Импортонезависимая криптография (ГОСТ Р 34.10-2012 / 34.11-2012 / 34.12-2015).

## 2. Условия применения

### 2.1. Условия, необходимые для выполнения программы

#### Технические условия
- iOS 15+ / Android 9+ / Windows 10+ / современный браузер с поддержкой Service Worker.
- Защищённое хранилище для секретов (Secure Enclave / Trusted Execution Environment).
- Биометрия — желательно, но не обязательно.
- Минимум 80 МБ свободного места.
- Сетевое подключение от 1 Мбит/с.

#### Программные условия
- Подтверждённая учётная запись Госуслуг (ЕСИА).
- Согласие на обработку персональных данных (152-ФЗ).

#### Условия безопасности
- Устройство не должно быть скомпрометировано (jailbreak/root → отказ работы).
- Должна быть включена защита экрана (PIN, биометрия, графический пароль).

### 2.2. Категории пользователей
- **Физические лица** — собственники недвижимости, транспорта, ИС, ценных бумаг.
- **Уполномоченные представители юридических лиц** — для управления активами организации.
- **Представители государственных предприятий** — для целей корпоративных программ.
- **Индивидуальные предприниматели, НКО, фонды** — собственники активов.

## 3. Описание задачи

Задача программы — обеспечить полный жизненный цикл взаимодействия собственника актива с Цифровой Платформой РСФСР:

1. Идентификация — регистрация через ЕСИА с подтверждением ПДн.
2. Сегментация — определение типа собственности.
3. Внесение актива — подача в систему с автоматической подгрузкой данных из государственных реестров.
4. Оценка — заявка авторизованному оценщику-партнёру.
5. Структурирование — настройка условий смарт-контракта в соответствии с целевой программой.
6. Подписание — электронная подпись усиленной квалифицированной ЭП.
7. Выпуск ЦФА — передача в Банк России.
8. Эмиссия — Целевое фондирование средств Банком России на эскроу.
9. Финансирование — перевод в Федеральное казначейство, реализация нацпроекта.
10. Мониторинг — отслеживание событий смарт-контракта.
11. Получение дохода — регулярные выплаты вознаграждения.
12. Мена — замена первоначального ЦФА на ЦФА построенного объекта.
13. Возврат — снятие обременения по истечении срока.
14. Реинвестирование (опционально) — новый цикл с тем же активом.

## 4. Входные и выходные данные

### 4.1. Входные данные
- ПДн пользователя из ЕСИА.
- Биометрические подтверждения из ЕБС.
- Действия пользователя (тапы, ввод).
- Данные из ЕГРН/Роспатента/депозитарных систем.
- Push-уведомления от серверной части.

### 4.2. Выходные данные
- API-запросы к серверной части ЦП РСФСР.
- Электронно подписанные документы (договоры, акты).
- Отчёты для пользователя (история, доходы, налоговые справки).

## 5. Эффективность применения

### 5.1. Социально-экономическая эффективность
- Повышение доступности механизма привлечения частного капитала в государственные проекты.
- Снижение трудоёмкости подачи и обслуживания залога в сравнении с традиционным банковским кредитованием на порядок.
- Сокращение времени от подачи актива до выпуска ЦФА с 30 дней (стандартный банковский залог) до 1–3 дней.
- Сохранение имущества в собственности и пользовании владельца с одновременным получением дохода.

### 5.2. Технико-экономические показатели
- Время на выпуск ЦФА: ≤ 30 секунд от подписания смарт-контракта.
- Доступность 24/7.
- Поддерживаемый одновременный объём пользователей: до 50 000 параллельно.
