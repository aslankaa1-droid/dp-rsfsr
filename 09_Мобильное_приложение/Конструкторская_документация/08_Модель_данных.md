## 1. Локальная модель данных (на устройстве)

### 1.1. Хранение

| Тип данных | Хранилище | Шифрование |
|---|---|---|
| Токены, ключи, биометрические идентификаторы | iOS Keychain / Android Keystore / Windows Credential Locker | средствами ОС |
| Настройки приложения | LocalStorage | нет (нечувствительные) |
| Кеш данных | IndexedDB / SQLite | AES-256 локальным ключом |
| Временные файлы | App Sandbox | средствами ОС |

## 2. Серверная модель данных

Полная модель данных описана в технической документации платформы (`01_Техническая_документация/06_DLT_реестр_ЦФА.md`). Клиент использует REST-проекции серверной модели.

### Ключевые сущности
- **Identity** (пользователь) — связь с ЕСИА.
- **KycRecord** — записи KYC-проверки.
- **Asset** — актив с привязкой к ЕГРН/Роспатенту/депозитарию.
- **Valuation** — оценка с партнёром, методом, страхованием.
- **SmartContract** — смарт-контракт с подписями всех сторон.
- **CFAToken** — токен в DLT-реестре с полной историей.
- **Escrow** — эскроу-запись Банка России.
- **Treasury** — записи Федерального казначейства.
- **NationalProgramme** — программа национального проекта.
- **InsurancePolicy** — страховой полис.
- **EgrnRecord** — запись обременения.
- **Notification** — уведомление пользователя.
- **AuditLog** — неизменяемый журнал всех операций.

## 3. Связи между сущностями

```
User ─< Asset ─< Valuation
        │
        └─< SmartContract ──> CFAToken ──> Escrow ──> Treasury
                                  │
                                  ├─> InsurancePolicy
                                  ├─> EgrnRecord
                                  └─< Transaction (выплаты)

User ─< Notification
User ─< AuditLog (read-only)
```

## 4. Жизненный цикл данных

### 4.1. Asset

```
DRAFT → PENDING_VALUATION → VALUED → CONTRACT_PENDING →
  ACTIVE → [RETURNED | REINVESTED | EXCHANGED]
```

### 4.2. CFA

```
PENDING_ISSUE → ISSUED → IN_CB_PORTFOLIO → ACTIVE →
  [MATURED → RETURNED | EXCHANGED_MENA | INSURED_CASE | REINVESTED]
```

| Сущность | Среднее количество на пользователя | Пиковое |

Совокупный объём локального кеша — до 200 МБ.

## 6. Резервное копирование

Данные пользователя автоматически реплицируются в серверной части. На устройстве хранится только кеш для оффлайн-режима. При смене устройства все данные восстанавливаются после первой авторизации в новом устройстве.

## 7. Удаление данных пользователя

При удалении аккаунта пользователя в соответствии с 152-ФЗ:
1. Данные на устройстве — удаляются сразу.
2. Данные в эксплуатационной БД — удаляются за 30 дней.
3. Данные в DLT-реестре (для активных ЦФА) — обременение должно быть снято; после этого реестр-запись остаётся в архивном виде с пометкой об удалении ПДн собственника, но с сохранением хронологии (требование надзорной отчётности).
4. Резервные копии — удаляются по графику ротации (до 7 лет хранения по требованию ЦБ).
