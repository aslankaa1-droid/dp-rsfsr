## 1. Назначение документа

Документ устанавливает требования к среде эксплуатации мобильного приложения «ЦП РСФСР» (далее — Приложение) на всех целевых платформах: Android, iOS, Windows, веб-PWA. Документ обязателен для:
- сертификации по ФСТЭК / ФСБ;
- категорирования объекта КИИ (ПП РФ №127);
- подачи в реестр операторов ИС ЦФА (Положение Банка России 779-П);
- развёртывания на платформах распространения (RuStore, App Store, Microsoft Store).

## 4. Требования к каналу связи

### 4.1. Пропускная способность

| Сценарий | Минимум | Рекомендация |
|---|---:|---:|
| Авторизация через ЕСИА (демо/прод) | 1 Мбит/с | 5 Мбит/с |
| Загрузка документов из ЕГРН (через СМЭВ 3) | 2 Мбит/с | 10 Мбит/с |
| Биометрическое подтверждение через ЕБС | 5 Мбит/с | 25 Мбит/с |
| Сценарий с просмотром портфеля | 1 Мбит/с | 5 Мбит/с |
| Подписание смарт-контракта УКЭП | 2 Мбит/с | 10 Мбит/с |
| Оффлайн-просмотр (только из локального кэша) | — | — |

### 4.2. Требования к качеству соединения

- Допустимая задержка (RTT) до серверного узла: не более 800 мс.
- Допустимая потеря пакетов: до 1%.
- Допустимые периоды без связи: оффлайн-режим Приложения сохраняет последние данные в локальном защищённом хранилище (Secure Storage / Keychain); синхронизация — при восстановлении связи через background sync.

### 4.3. Безопасность канала связи

- Исключительно HTTPS (TLS 1.2+); cleartext запрещён (`allowMixedContent: false`).
- Certificate Pinning с двумя пинами (primary + backup) для домена `dp-rsfsr.ru` (см. `Сборки/android/README.md`, `Сборки/ios/README.md`).
- Поддержка ГОСТ TLS (CryptoPro CSP) для интеграции с СКЗИ-каналами Банка России.

## 5. Требования к ЦОД-инфраструктуре оператора

### 5.1. Целевая инфраструктура (требуется для production)

| Параметр | Значение |
|---|---|
| Количество ЦОД (геораспределённое размещение) | ≥ 3 (включая один holding center) |
| Уровень ЦОД (Uptime Institute / ГОСТ Р 51275) | Tier-III или эквивалент |
| Резервирование питания | N+1 (по UPS и генераторам) |
| Резервирование охлаждения | N+1 |
| Резервирование сетевых каналов | 2 независимых провайдера |
| Минимальная пропускная способность каналов | 10 Гбит/с с возможностью масштабирования |
| Целевой uptime | 99,95% (≈ 4 ч простоя в год) |
| RTO (Recovery Time Objective) | ≤ 4 часа |
| RPO (Recovery Point Objective) | ≤ 15 минут |

### 5.2. Категорирование размещения

- **Российская юрисдикция** — все ЦОД физически размещены на территории РФ (соответствие ст. 12 152-ФЗ о локализации ПДн граждан РФ).
- **Аттестация ФСТЭК** — все площадки аттестованы по требованиям к значимым объектам КИИ (К1) согласно Приказу ФСТЭК №239.
- **Уровень защищённости УЗ-1** по ПП РФ №1119.
- **Конкретные операторы ЦОД** — выбор по тендеру; целевые кандидаты на момент подготовки: «Ростелеком-ЦОДы», DataPro, ИКС-Холдинг, IXcellerate. Финальный выбор фиксируется при заявке в реестр операторов ИС ЦФА (779-П).

### 5.3. DLT-инфраструктура

- DLT-вендор: **Мастерчейн** (целевой; альтернатива при недоступности — Waves Enterprise) — конкретный выбор фиксируется в заявке Банку России.
- Тип реестра: permissioned (с регулятор-узлом ЦБ).
- Финальность: детерминированная.
- Криптография: ГОСТ Р 34.10-2012 (подписи), ГОСТ Р 34.11-2012 (хеш), ГОСТ Р 34.12-2015 (шифр).
- Регулятор-узел ЦБ: предусмотрен в архитектуре (см. РСФСР.МП.06 «Описание архитектуры», п. 3.7).

## 6. Требования к окружающей среде физических устройств

### 6.1. Климатические условия эксплуатации

Стандартные условия для мобильных и стационарных устройств конечного пользователя:
- Температура: от 0 °C до +40 °C.
- Влажность: до 80% (без конденсации).
- Атмосферное давление: 84–106,7 кПа.

### 6.2. Условия эксплуатации серверного оборудования

- Температура: 18–27 °C (соответствует ASHRAE TC 9.9 рекомендациям).
- Влажность: 40–60% (без конденсации).
- Защита от пыли и электромагнитных помех — по нормативам Tier-III ЦОД.

## 8. Требования к режиму работы и обслуживанию

### 8.1. Режим работы Приложения

- **Конечный пользователь:** 24×7 (с плановыми технологическими окнами).
- **Бэкенд оператора:** 24×7 с целевым uptime 99,95%.
- **Плановое технологическое окно:** ночь воскресенья 02:00–06:00 МСК (1 раз в месяц); анонс не позднее 7 дней до окна.
- **Аварийное технологическое окно:** допустимо до 2 часов с предварительным уведомлением.

### 8.2. Регламент обновлений

- Мобильное приложение: автообновление через RuStore / App Store / Microsoft Store; критические security-патчи — в течение 72 часов после обнаружения.
- Серверная часть: blue-green deployment, без простоя.

### 8.3. Резервное копирование

- Полное еженощное резервирование баз данных.
- Хранение бэкапов — в отдельной геолокации (≥ 500 км от основного ЦОД).
- Криптографическое шифрование бэкапов ГОСТ-алгоритмами.
- Регулярное тестирование восстановления (DR-drill) — не реже 1 раза в квартал.

## 9. Особые требования

### 9.1. Безопасность среды эксплуатации

- Конечное устройство пользователя должно поддерживать аппаратный TEE (Trusted Execution Environment) для хранения криптографических ключей пользователя. Минимальные требования:
  - Android: Strongbox / TEE Keymaster (API 28+).
  - iOS: Secure Enclave.
  - Windows: TPM 2.0.
- На устройствах с jailbreak / root приложение работает в ограниченном режиме (только просмотр истории) с предупреждением.

### 9.2. Чувствительность к качеству среды

- При снижении уровня TLS ниже 1.2 — отказ от соединения.
- При обнаружении MITM-атаки (несоответствие pinned сертификата) — блокировка с уведомлением CISO.
- При деградации канала ниже 1 Мбит/с — автоматическое переключение в lite-режим (только текстовые данные).
