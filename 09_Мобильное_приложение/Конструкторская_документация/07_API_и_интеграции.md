# API и интеграции

## ЦП РСФСР — мобильное приложение

**Условное обозначение:** РСФСР.МП.07.API

---

## 1. Принципы взаимодействия

- REST-архитектура, JSON-форматы.
- Аутентификация: OAuth 2.1 (PKCE) + JWT с ГОСТ-подписью.
- TLS 1.3 с ГОСТ-сертификатами (КриптоПро TLS).
- mTLS для критичных операций.
- Rate limiting на стороне сервера: 100 запросов в минуту на пользователя.

## 3. Аутентификация

### POST /auth/esia/init
Инициация авторизации через ЕСИА.

**Запрос:**
```json
{
  "redirect_uri": "ru.rsfsr.app://auth/callback",
  "code_challenge": "...",
  "code_challenge_method": "S256"
}
```

**Ответ:**
```json
{
  "authorize_url": "https://esia.gosuslugi.ru/aas/oauth2/v3/ac?..."
}
```

### POST /auth/esia/callback
Завершение авторизации через ЕСИА.

**Запрос:**
```json
{ "code": "...", "code_verifier": "..." }
```

**Ответ:**
```json
{
  "access_token": "...",
  "refresh_token": "...",
  "expires_in": 900,
  "user": { "id": "...", "fio": "..." }
}
```

### POST /auth/refresh
Обновление access-токена.

### POST /auth/biometric
Авторизация через ЕБС.

### POST /auth/logout
Завершение сессии.

## 5. Активы

### GET /assets
Список активов пользователя с фильтрацией и пагинацией.

**Параметры запроса:**
- `type` — тип актива (real, land, vehicle, ip, securities).
- `status` — статус (draft, valuation, active, returned).
- `page`, `limit`.

### POST /assets
Подача нового актива на залог.

**Запрос:**
```json
{
  "type": "real",
  "kadastr": "77:01:0001000:1234",
  "address": "...",
  "area": 97,
  "year": 2015,
  "ownership_doc_type": "...",
  "ownership_doc_attachments": ["doc-uuid-1"]
}
```

### GET /assets/{id}
Получение детальной информации об активе.

### POST /assets/{id}/valuation
Запрос на оценку актива.

### GET /assets/{id}/valuation
Получение результата оценки.

### DELETE /assets/{id}
Отзыв заявки (только для статуса draft).

## 6. Смарт-контракты

### POST /contracts
Создание смарт-контракта на основе оценённого актива.

**Запрос:**
```json
{
  "asset_id": "...",
  "term_months": 36,
  "target_programme": "...",
  "rate_pct": 4.2,
  "nominal": 8200000
}
```

### GET /contracts/{id}
Получение текста смарт-контракта в формате PDF и его параметров.

### POST /contracts/{id}/sign
Подписание смарт-контракта.

**Запрос:**
```json
{
  "signature_method": "uke",
  "uke_token": "...",
  "otp_code": "742369"
}
```

### POST /contracts/{id}/cancel
Отзыв подписанного контракта (только до выпуска ЦФА).

## 12. Интеграции с государственными системами

### 12.1. ЕСИА (Госуслуги)
- **Стандарт:** OAuth 2.1 + OpenID Connect 1.0.
- **Уровень учётной записи:** Подтверждённая (УЗ-3).
- **Получаемые scope:** `openid`, `fullname`, `birthdate`, `gender`, `snils`, `inn`, `id_doc`, `email`, `mobile`.

### 12.2. Единая биометрическая система
- **Стандарт:** Положение Банка России №482-П.
- **Типы биометрии:** лицо, голос.
- **Назначение:** идентификация и подтверждение операций.

### 12.3. ЕГРН (Росреестр)
- **Стандарт:** Регламент электронного взаимодействия с Росреестром (через серверную часть платформы).
- **Сценарии:** получение сведений об объекте по кадастровому номеру; регистрация и снятие обременений.

### 12.4. Госуслуги (личный кабинет)
- **Назначение:** получение официальных уведомлений; передача согласий на обработку ПДн.
- **Сценарии:** уведомления об обременении, мене, выплатах.

### 12.5. НСПК (Мир)
- **Назначение:** зачисление выплат на карту «Мир».

## 13. Форматы ошибок

Все ошибки возвращаются в едином формате:

```json
{
  "error": {
    "code": "ASSET-01",
    "message": "Объект не найден в ЕГРН",
    "details": {
      "kadastr": "..."
    },
    "trace_id": "..."
  }
}
```

## 14. Версионирование API

- Версия в URL: `/v1/`, `/v2/`, ...
- Поддержка обратной совместимости в течение 12 месяцев после релиза новой версии.
- Уведомления о deprecation через response headers.
