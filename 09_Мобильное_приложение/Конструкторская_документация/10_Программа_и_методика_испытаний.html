Программа и методика испытаний

Программа и методика испытаний

ЦП РСФСР — мобильное приложение

Условное обозначение: РСФСР.МП.10.ПМИ Стандарт: ГОСТ 19.301-79

1. Объект испытаний

1.1. Наименование

Мобильное приложение «ЦП РСФСР — мобильное приложение залогодателя» во всех целевых сборках: - Android (RuStore, Google Play); - iOS (App Store); - Windows (Microsoft Store); - PWA (любой современный браузер).

1.2. Цель испытаний

Подтверждение соответствия программы требованиям технического задания (РСФСР.МП.01.ТЗ), нормативным требованиям регуляторов и стандартам качества программных продуктов.

2. Виды испытаний

2.1. Функциональные испытания

Проверка соответствия функциональным требованиям ТЗ. Объект — все 23 экрана и все переходы.

2.2. Нагрузочные испытания

Проверка производительности при пиковых нагрузках.

2.3. Испытания безопасности

Сертификационные испытания по требованиям ФСТЭК России, ФСБ России.

2.4. Испытания доступности

Проверка соответствия WCAG 2.2 уровня AA.

2.5. Испытания совместимости

Проверка работы на различных моделях устройств и версиях ОС.

2.6. Регрессионные испытания

Автоматические тесты на каждом релизе.

3.2. Этап 2. Бета-тестирование (закрытое)

Закрытый круг пользователей до 1 000 человек, в течение 30 дней. Метрики: - Количество завершённых пользовательских сценариев. - Среднее время прохождения каждого сценария. - Количество ошибок на 100 операций. - Crash-free sessions (целевое: ≥ 99,8%).

3.3. Этап 3. Приёмочные испытания

С участием заказчика, регулятора (Банк России), представителей профильных министерств. Проверка соответствия всем требованиям ТЗ.

3.4. Этап 4. Сертификационные испытания

Сертификация СЗИ

  • Аттестация по требованиям приказа ФСТЭК России №239 (КИИ).
  • Аттестация по требованиям приказа ФСТЭК России №21 (ПДн, УЗ-1).
  • Соответствие ГОСТ Р 57580.1 уровня 4.
  • Сертификация криптографических средств ФСБ России.

Сертификация ПО

  • Включение в Единый реестр российского ПО (ПП РФ №1236).

4. Методика испытаний

4.1. Функциональные испытания

Подготовка

  1. Создаётся тестовое окружение с моками внешних систем (ЕСИА, ЕБС, ЕГРН, ЦБ).
  2. Подготавливается набор тестовых пользователей с разными KYC-статусами.
  3. Подготавливаются тестовые активы и ЦФА в различных состояниях.

Проведение

  • Автоматические тесты через Cypress / Playwright для PWA-режима.
  • Capacitor Test Kit (Appium-обвязка над нативным wrapper'ом) для iOS/Android-сборок.
  • WebView2 Test Harness для Windows-сборки.
  • XCUITest для iOS native-расширений (биометрия, Secure Enclave, ГОСТ-канал).
  • Espresso для Android native-расширений (Strongbox, BiometricPrompt, СКЗИ).

Критерии успеха

  • 100% покрытие сценариев FT-001 — FT-020.
  • 0 критических ошибок.
  • Не более 5 косметических замечаний.

4.2. Нагрузочные испытания

Подготовка

  • Тестовая среда, аналогичная продакшну.
  • JMeter / Yandex Tank для серверной нагрузки.
  • Lighthouse / WebPageTest для клиентских метрик.

Сценарии

  • 5 000 RPS на API-шлюзе в течение 30 минут.
  • 50 000 одновременных пользователей.
  • Регистрация 1 000 новых пользователей за 5 минут.
  • Выпуск 100 ЦФА за час.

Критерии успеха

  • Время отклика P95 ≤ 200 мс.
  • Error rate ≤ 0,1%.
  • Cold start ≤ 2 секунды на 4G.
  • Memory peak ≤ 300 МБ.

4.3. Испытания безопасности

Виды проверок

  • Статический анализ кода (SAST): PT Application Inspector, PVS-Studio.
  • Динамический анализ (DAST): PT BlackBox.
  • Анализ компонентов (SCA): обнаружение уязвимых библиотек.
  • Внешний пентест аккредитованной лабораторией.
  • Внутренний Red Team.
  • Проверка на соответствие OWASP MASVS L2+.

Критерии успеха

  • 0 уязвимостей категорий Critical и High.
  • Не более 5 уязвимостей Medium с принятым риском.
  • Подтверждение соответствия ГОСТ Р 57580.1 уровня 4.

4.4. Испытания доступности

Виды проверок

  • Автоматическая проверка через axe-core / WAVE.
  • Тестирование с экранными читалками (VoiceOver, TalkBack, Narrator).
  • Тестирование с увеличенным шрифтом.
  • Проверка контрастности (≥ 4,5:1 для основного текста).

Критерии успеха

  • Соответствие WCAG 2.2 уровня AA на 100%.
  • Все интерактивные элементы доступны с клавиатуры.
  • Все изображения имеют alt-атрибуты.

4.5. Испытания совместимости

Парк устройств

Android: - Samsung Galaxy S22, S23, S24. - Xiaomi Redmi Note 12, 13. - Realme C55, C65. - Honor 90, X9. - Pixel 7, 8.

iOS: - iPhone SE (3-го поколения). - iPhone 13, 14, 15, 16. - iPad 9-го поколения, iPad Pro.

Windows: - Surface Pro 9, 10. - Типовые рабочие станции с Windows 10/11. - Surface Go (ARM64).

Критерии успеха

  • Корректная работа на каждом устройстве.
  • Адаптивная вёрстка от 360 × 640 до 2560 × 1600.

5.1. Состав отчёта
