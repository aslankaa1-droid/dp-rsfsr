Описание архитектуры мобильного приложения

ЦП РСФСР — мобильное приложение

Условное обозначение: РСФСР.МП.06.ОА

1. Архитектурный обзор

1.1. Подход

Приложение построено по принципу PWA-first с упаковкой в нативные оболочки через Capacitor 6. Это обеспечивает единую кодовую базу для веб, Android, iOS и Windows при сохранении доступа к нативным API каждой платформы.

1.2. Преимущества подхода

  • Единая кодовая база — экономия 60–70% ресурсов разработки.
  • Возможность независимого выпуска платформ.
  • Полноценная PWA для пользователей без установки.
  • Доступ к нативным API через плагины Capacitor.
  • Соответствие требованиям импортозамещения (без зависимости от иностранных кросс-платформенных фреймворков).

3. Компонентная структура

3.1. Presentation Layer

  • Экраны — 23 экрана как отдельные секции HTML с уникальными атрибутами data-screen.
  • Компоненты — переиспользуемые UI-элементы (карточки, кнопки, формы, графики).
  • Темы — четыре темы оформления через CSS Custom Properties.
  • i18n — поддержка четырёх языков через JSON-словари.

3.2. Application Layer

  • Router — навигация по хешу URL (#screen-name). Поддержка истории браузера и кнопки «Назад» ОС.
  • State Manager — централизованное хранилище состояния пользователя, активов, ЦФА, истории, уведомлений.
  • Theme Manager — переключение тем с сохранением в LocalStorage.
  • i18n Manager — локализация интерфейса с автоматическим переключением направления текста для арабского.
  • A11y Manager — управление размером шрифта, контрастной темой, фокусом.

3.3. Domain Layer

Бизнес-сущности: - User — пользователь с идентификатором, ФИО, KYC-статусом. - Asset — актив с типом, характеристиками, оценкой, статусом. - Valuation — оценка с методом, ценой, сроком действия, страхованием. - SmartContract — смарт-контракт с условиями, целью, ставкой, статусом. - CFA — цифровой финансовый актив с привязкой к смарт-контракту, выплатам. - Transaction — операция в реестре (выпуск, мена, выплата). - Notification — уведомление с категорией, статусом, действием.

3.4. Data Layer

  • Repositories — абстракция доступа к данным с переключением между локальным хранилищем и сетевым API.
  • API Client — типизированный клиент REST API с retry-логикой и кешированием.
  • Local Storage — LocalStorage для настроек, IndexedDB для кеша данных, защищённое хранилище ОС для секретов.

3.5. Infrastructure

  • Service Worker — кеширование статических ресурсов и API-ответов; обработка push-уведомлений; работа в оффлайн-режиме.
  • Crypto — обёртка над WebCrypto API + нативный плагин для ГОСТ-криптографии (КриптоПро Mobile SDK).
  • Network — Fetch API c автоматическим обновлением токенов; certificate pinning через нативные плагины.

6. Информационная безопасность

6.1. Защита приложения

  • Anti-tampering — проверка целостности кода при запуске.
  • Anti-debug — обнаружение отладчиков и эмуляторов.
  • Jailbreak/Root detection — отказ работы на скомпрометированных устройствах.
  • Certificate pinning — закрепление сертификатов CA для защиты от MITM.
  • Anti-screenshot — для критичных экранов (подписание, смарт-контракт).
  • Obfuscation — обфускация JS-кода при сборке.

6.2. Защита данных

  • Шифрование at rest — все данные на устройстве шифруются ключом из защищённого хранилища.
  • Шифрование in transit — TLS с ГОСТ-сертификатами.
  • Нет хранения биометрии — только идентификаторы для верификации через ЕБС.
  • Token rotation — обновление токенов авторизации каждые 15 минут.

6.3. Соответствие стандартам

  • OWASP MASVS L2+.
  • ГОСТ Р 57580.1 уровень 4.
  • 152-ФЗ (защита персональных данных).
  • 187-ФЗ (КИИ, категория 1).
  • Положение Банка России №683-П.

7.2. Оптимизации

  • Lazy loading экранов и компонентов.
  • Минификация и tree-shaking JS.
  • Сжатие изображений (WebP).
  • Кеширование API-ответов через Service Worker.
  • Использование requestIdleCallback для некритичных задач.

8. Доступность (Accessibility)

  • WCAG 2.2 уровня AA (обязательно), AAA (целевое).
  • Поддержка VoiceOver, TalkBack, Narrator.
  • Динамическое увеличение шрифта.
  • Контрастная тема для слабовидящих.
  • Минимальный размер сенсорной зоны 44 × 44 пикселя.
  • Корректные ARIA-атрибуты для всех интерактивных элементов.
  • Skip links для быстрой навигации.

9. Локализация

Приложение поддерживает четыре языка: - Русский (по умолчанию). - Английский. - Французский. - Арабский (с автоматическим переключением на RTL-разметку).

Все строки извлечены в централизованный JSON-словарь (_shared/i18n.js). Добавление нового языка — добавление одного объекта в словарь.

10. Развёртывание

10.1. Сборка и упаковка

Подробное описание процесса сборки приведено в README соответствующих платформ: - Сборки/android/README.md - Сборки/ios/README.md - Сборки/windows/README.md

10.2. CI/CD

Рекомендуемая схема непрерывной интеграции/доставки: - Git-репозиторий на отечественной платформе (GitFlic / GitVerse). - CI на каждом коммите: lint, unit-тесты, SAST-сканирование, сборка. - CD по тегам версий: автоматическая публикация в RuStore и другие сторы. - Распределение бета-сборок через RuStore Beta и TestFlight.
