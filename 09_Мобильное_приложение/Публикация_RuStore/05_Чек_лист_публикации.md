# Чек-лист публикации в RuStore

## ЦП РСФСР — мобильное приложение

---

## Этап 1. Подготовка разработчика

- [ ] Зарегистрирован аккаунт в RuStore Console: https://console.rustore.ru
- [ ] Подтверждена учётная запись через Госуслуги (для российских разработчиков).
- [ ] Загружены документы:
  - [ ] паспорт или иной документ для физического лица;
  - [ ] учредительные документы для юридического лица;
  - [ ] свидетельство ИНН.
- [ ] Подписано соглашение разработчика RuStore.

## Этап 2. Подготовка приложения

### Технические артефакты
- [ ] AAB-файл собран (`app-release.aab`).
- [ ] Размер AAB ≤ 150 МБ (лимит RuStore).
- [ ] AAB подписан release-keystore (не debug-keystore).
- [ ] versionCode и versionName согласованы.
- [ ] minSdkVersion = 28 (Android 9).
- [ ] targetSdkVersion = 34 (Android 14).
- [ ] applicationId = `ru.rsfsr.app`.

### Соответствие
- [ ] Приложение работает на минимальной целевой версии Android (Android 9).
- [ ] Приложение тестировалось минимум на 5 разных устройствах.
- [ ] Crash-rate < 0,5% по данным внутренних тестов.

### Контент
- [ ] Иконка 512 × 512 PNG.
- [ ] Feature graphic 1024 × 500 PNG.
- [ ] Минимум 5 скриншотов (1080 × 1920 рекомендуется).
- [ ] Промо-видео (опционально, но рекомендуется).
- [ ] Краткое описание (до 80 символов).
- [ ] Полное описание (до 4000 символов).
- [ ] Текст «Что нового» (до 500 символов).
- [ ] Ключевые слова.

### Правовые документы
- [ ] Политика конфиденциальности опубликована по адресу `https://dp-rsfsr.ru/privacy` (или временный домен).
- [ ] Условия использования опубликованы по адресу `https://dp-rsfsr.ru/terms`.
- [ ] Адрес сайта разработчика подтверждён.
- [ ] Контактный email поддержки активен.

## Этап 3. Заполнение карточки в RuStore Console

### Основные сведения
- [ ] Название: «ЦП РСФСР».
- [ ] Подзаголовок: «Российский Сервис Финансирования Социального Развития».
- [ ] Категория: Финансы.
- [ ] Подкатегория: Инвестиции.
- [ ] Возрастная категория: 18+.
- [ ] Цена: Бесплатно.

### Описание
- [ ] Краткое описание заполнено.
- [ ] Полное описание заполнено.
- [ ] Ключевые слова указаны.
- [ ] «Что нового» заполнено.

### Графика
- [ ] Иконка загружена.
- [ ] Feature graphic загружен.
- [ ] 5–8 скриншотов загружены.
- [ ] Промо-видео загружено (если есть).

### Технические сведения
- [ ] AAB файл загружен.
- [ ] Маппинг (mapping.txt для ProGuard) загружен.
- [ ] Подписи в порядке (не используется debug-keystore).

### Контент-рейтинг
- [ ] Заполнена анкета IARC / российская система маркировки.
- [ ] Установлена возрастная категория 18+.

### Локализация
- [ ] Русский язык — основной, заполнен полностью.
- [ ] Английский язык — заполнен.
- [ ] Французский язык — заполнен (опционально).
- [ ] Арабский язык — заполнен (опционально).

### Правовая информация
- [ ] Ссылка на Политику конфиденциальности.
- [ ] Ссылка на Условия использования.
- [ ] Контактные данные разработчика.
- [ ] Указано, что приложение не использует рекламу.
- [ ] Указано, что приложение не имеет внутренних покупок.

### Финансовая информация (если применимо)
- [ ] Указан тип финансовой деятельности.
- [ ] Загружены копии лицензий (Банк России — после получения).
- [ ] Указана связь с государственными системами (Госуслуги, ЕГРН).

## Этап 4. Безопасность и регуляторика

### Перед публикацией production
- [ ] Получен сертификат ФСТЭК России (К1 / КИИ-1).
- [ ] Получен сертификат ФСБ России (СКЗИ).
- [ ] Получено уведомление Роскомнадзора о включении в реестр операторов ПДн.
- [ ] Приложение включено в Единый реестр российского ПО (ПП РФ №1236).
- [ ] Оператор включён в реестр операторов ИС Банка России (Положение №779-П).

### Альтернативно: для пилотного режима
- [ ] Получено разрешение Банка России на пилотную эксплуатацию.
- [ ] В приложении явно указан статус «Пилотный режим» (в About-разделе).
- [ ] Доступ ограничен пилотным пулом пользователей через серверную часть.

## Этап 5. Тестирование

### Внутреннее тестирование
- [ ] Альфа-тестирование пройдено (≥ 50 пользователей).
- [ ] Все критические сценарии (20 тестов) пройдены.
- [ ] Найденные ошибки устранены.

### Бета-тестирование
- [ ] Бета-сборка распространена через RuStore Beta или внутренний канал.
- [ ] Получены отзывы (≥ 100 пользователей).
- [ ] Crash-free sessions ≥ 99,8%.
- [ ] Найденные ошибки устранены.

## Этап 6. Подача на модерацию

- [ ] Все поля заполнены.
- [ ] Все артефакты загружены.
- [ ] Запрос на проверку отправлен в RuStore Console.

### Срок рассмотрения
**1–3 рабочих дня** (стандартный).

### Возможные итоги
- ✓ Одобрено — публикация в каталоге.
- ⚠ Доработать — комментарии модератора → внесите исправления → повторная подача.
- ✗ Отклонено — критическое нарушение → анализ → переработка → повторная подача.

## Этап 7. После публикации

### Мониторинг
- [ ] Мониторинг загрузок в RuStore Console.
- [ ] Мониторинг отзывов пользователей.
- [ ] Crash reporting подключен (AppMetrica / отечественная альтернатива).
- [ ] Мониторинг основных KPI (DAU, MAU, retention).

### Поддержка
- [ ] Команда поддержки готова отвечать на обращения.
- [ ] Email support@dp-rsfsr.ru мониторится 7×24.
- [ ] Чат-бот в приложении работает.

### Обновления
- [ ] Расписание регулярных обновлений (1–2 раза в месяц).
- [ ] Hot-fix процедура для критических уязвимостей.
- [ ] Обратная совместимость API ≥ 12 месяцев.
