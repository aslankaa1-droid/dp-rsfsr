مواصفات تطبيق الهاتف المحمول

Автор: Кагиров А.-Х.А. · Свидетельство о депонировании №4011265 от 19.12.2024 · Center Group Company · Engagement: CPFSR-RF-DESIGN-2026

مواصفات تطبيق CPFSR للهاتف المحمول

المنصّات: iOS / Android / Windows. الجمهور المستهدف: الراهنون — الأشخاص الطبيعيون والاعتباريون، ممثلو المؤسسات الحكومية، رواد الأعمال الأفراد، المنظمات غير الربحية، والمؤسسات. الأهداف الرئيسية: توفير إدارة آمنة عن بُعد للأصول، وإصدار DFA وخدمتها، واستلام المكافأة.

1. حزمة التقنيات

1.1. التطبيق متعدد المنصات

  • المنصة الأساسية: PWA (HTML/CSS/JS + TypeScript)، Service Worker لوضع عدم الاتصال والتحديث في الخلفية، Manifest مع shortcuts و share_target.
  • غلاف iOS / Android: Capacitor v6+ (أغلفة iOS/Android أصلية فوق نواة PWA)؛ قاعدة شيفرة موحَّدة، وصول أصلي للقياسات الحيوية، ووسائل التشفير وفق GOST، وSecure Enclave/Strongbox، وcertificate pinning، وإشعارات push، وshare_target.
  • Windows سطح المكتب: PWA عبر Edge WebView2 (تثبيت منفصل عبر Microsoft Store / MSIX) أو edge_side_panel.
  • منطق الأعمال المشترك: TypeScript + وحدات WebAssembly للوظائف الحرجة (التشفير، التوقيع).
  • نظام التصميم: قائم على رموز هوية المشروع (_shared/brand.css)، مع تجاوزات أصلية لـ iOS (SF Pro) وAndroid (Roboto).

1.2. بنية تطبيق العميل

  • النمط: PWA Shell + شاشات تُحمَّل عند الحاجة (lazy-loaded).
  • إدارة الحالة: state-store داخلي في app.js (vanilla JS) أو محوّل Zustand مع تنامي التعقيد.
  • التنقّل: back-stack PWA عبر History API + drawer-aside في الأوضاع كاملة الحجم.
  • عميل API: توليد تلقائي من OpenAPI 3.1.

1.3. تخزين البيانات على الجهاز

  • البيانات الحساسة: iOS Keychain / Android Keystore / Windows Credential Locker — رموز مشفَّرة فقط.
  • لا تُخزَّن البيانات البيومترية محلياً — فقط معرّفات للتحقق عبر EBS.
  • ذاكرة التخزين المؤقت المحلية — SQLCipher (SQLite مشفَّر).

1.4. التشفير

  • UKEP في الجهاز المحمول — عبر KriptoPro EP (SDK المحمول).
  • التجزئة — GOST R 34.11-2012.
  • قنوات TLS — باستخدام شهادات GOST.

2. الأمن

2.1. التعريف والمصادقة

  1. الترخيص الأوّلي — عبر ESIA (OpenID Connect) بحساب مؤكَّد.
  2. عوامل إضافية:
    • بيومترية الجهاز (TouchID / FaceID / Windows Hello) للوصول إلى التطبيق.
    • بيومترية عبر EBS (نظام بنك روسيا رقم 482-P) لتوقيع العمليات الحرجة.
    • OTP (SMS / push) للعمليات الحرجة (توقيع العقد الذكي).

2.2. حماية التطبيق

  • Anti-tampering: Mobile Application Shielding (MAS) — حلول محلية الفئة.
  • كشف Jailbreak / Root — رفض العمل على الأجهزة المخترَقة.
  • Anti-debug — تشويش، حماية من حقن وقت التشغيل.
  • Anti-screenshot للشاشات الحرجة (العقد الذكي، OTP).
  • Certificate pinning على قنوات TLS.

2.3. الحماية من الاحتيال

  • التحليل السلوكي (البيومترية السلوكية).
  • Device fingerprinting.
  • تحديد الموقع الجغرافي والأنماط الزمنية.

2.4. تخزين الأسرار

  • فقط في TEE (Trusted Execution Environment) / Secure Enclave.
  • لا مفاتيح في الذاكرة المشتركة للتطبيق.

2.5. الامتثال

  • FZ-152 (حماية البيانات الشخصية).
  • FZ-187 (CII) — التطبيق جزء من كائن CII مهم من الفئة 1.
  • تعليمة بنك روسيا رقم 6635-U (حوادث أمن المعلومات).
  • معيار OWASP MASVS Level 2+.
  • المطابقة لـ WCAG 2.2 AA (إمكانية الوصول).

3. الوحدات الوظيفية

3.1. التسجيل والتعريف

  • الترخيص عبر ESIA.
  • التحقق البيومتري عبر EBS.
  • ربط الجهاز.
  • موافقات معالجة البيانات الشخصية.

3.2. الحساب الشخصي

  • لوحة المعلومات: الرصيد الإجمالي للأصول العاملة، الدخل الحالي، محفظة DFA.
  • قائمة الأصول مع فلاتر وبحث.
  • لكل أصل: الخصائص، التاريخ، DFA الحالي.

3.3. تقديم الأصل كضمان

  • اختيار نوع الملكية.
  • تحميل البيانات من السجلات الحكومية (EGRN، Rospatent، جهات الإيداع).
  • إرفاق المستندات.
  • طلب التقييم.

3.4. التقييم

  • عرض حالة الطلب.
  • نتيجة التقييم مع إمكانية الطعن.
  • معلومات التأمين.

3.5. العقد الذكي

  • مصمِّم الشروط.
  • عرض النص الكامل للعقد.
  • التوقيع الإلكتروني بـ UKEP (شهادة المحمول).

3.6. إدارة DFA

  • قائمة DFA النشطة.
  • حالة كل DFA في سجل DLT.
  • الأحداث المتوقعة (التبادل، حالات التأمين، انتهاء المدة).

3.7. الدخل

  • سجل المدفوعات.
  • ملخص ربع سنوي وسنوي.
  • تنزيل شهادة 2-NDFL.

3.8. الإشعارات

  • إشعارات push للأحداث الحرجة.
  • إشعارات عبر Gosuslugi (للموافقات والتعديلات).

3.9. الملف الشخصي

  • إدارة بيانات الاتصال.
  • إدارة الأمن (البيومترية، قناة OTP).
  • إدارة الموافقات.
  • بيانات الدفع.

3.10. المساعدة والدعم

  • FAQ.
  • روبوت محادثة.
  • التواصل مع موظف الدعم.

5. خطة الإصدارات

5.1. iOS / Android

  • إصدار تجريبي عبر TestFlight / Google Play Internal Testing — الشهر 10.
  • اختبار مغلق — الشهر 12.
  • الإطلاق المفتوح — الشهر 18 (بالتزامن مع المنصة).
  • النشر في RuStore (وزارة التطوير الرقمي).

5.2. Windows / Desktop (Capacitor v6+ لسطح المكتب / PWA installable)

  • إصدار تجريبي — الشهر 14 (PWA عبر Edge / Chrome «Install app»؛ حزمة Capacitor Windows كاملة — الشهر 16).
  • الإطلاق المفتوح — الشهر 18.
  • النشر في Microsoft Store (عبر حزمة MSIX التي يولِّدها Capacitor) + قنوات بديلة (web-distribution PWA).
  • ميزة قاعدة الشيفرة الموحَّدة لـ PWA: المصادر نفسها كما لـ iOS/Android — لا توجد حزمة .NET MAUI متوازية.

5.3. التحديثات

  • التحديث التلقائي عبر المتاجر القياسية.
  • تحديثات إلزامية للثغرات الحرجة عبر آلية داخل التطبيق.

6. الاختبار

6.1. أنواع الاختبار

  • اختبارات الوحدة (Vitest + @testing-library/dom لمنطق PWA، JUnit/XCTest لإضافات Capacitor الأصلية) — تغطية ≥ 80%.
  • اختبارات التكامل (Playwright Component Testing) — السيناريوهات الحرجة.
  • اختبارات E2E (Playwright cross-browser + Capacitor + Appium للأغلفة الأصلية) — التدفقات الأساسية للمستخدم لـ iOS / Android / desktop.
  • اختبارات الأداء (Lighthouse CI ≥ 90 على المقاييس الأربعة لـ PWA) على أجهزة مختلفة.
  • اختبارات الأمن (OWASP MASVS L2+ FT-021..025، ZAP / Burp، OWASP Mobile Top 10، فاحصات معتمدة من FSTEC).
  • اختبارات إمكانية الوصول (axe-core، WCAG 2.2 AA، قارئات الشاشة iOS VoiceOver / Android TalkBack).

6.2. أسطول الأجهزة

  • iOS: iPhone 13 / 14 / 15 / 16؛ iPad 9-gen / Pro.
  • Android: Samsung Galaxy S22+، Xiaomi Redmi Note 12، Realme C55 (الشريحة الاقتصادية).
  • Windows: Surface Pro 9، محطات عمل قياسية.

6.3. CI/CD

  • البناء في بيئة مؤمَّنة مع توقيع UKEP.
  • الاختبار التلقائي عند كل دمج.
  • فروع الإصدارات + الوسوم.

7. المطابقة لمتاجر التطبيقات

7.1. RuStore (الأولوية)

  • مطابقة كاملة لمتطلبات RuStore.
  • الإيداع في المتجر خلال الـ 90 يوماً الأولى بعد الإطلاق العام.

7.2. App Store

  • المطابقة لقواعد Apple.
  • قيود محتملة على العملات المشفرة — DFA ليست عملات مشفرة؛ قواعد Apple بشأن الأصول المالية المرمَّزة — تسمح، رهناً بتوفّر الترخيص.

7.3. Google Play

  • المطابقة لسياسة Play.
  • الإيداع في فئة التطبيقات المالية مع تأكيد التسجيل في الاتحاد الروسي.

7.4. Microsoft Store

  • المطابقة لسياسة Store.
  • فئة Finance / Productivity.

8. التوطين وإمكانية الوصول

8.1. التوطين

  • اللغة الأساسية: الروسية.
  • دعم كامل للأبجدية السيريلية في جميع الشاشات.
  • توطين التواريخ والأرقام والعملات.
  • اللغة الإنجليزية — للجمهور الدولي (اختياري).

8.2. إمكانية الوصول

  • WCAG 2.2 AA.
  • التحكم الصوتي عبر VoiceOver / TalkBack / Narrator.
  • التكبير الديناميكي للخطوط.
  • تباين عالٍ.
