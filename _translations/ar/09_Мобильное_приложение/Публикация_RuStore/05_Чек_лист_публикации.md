# قائمة التحقق للنشر في RuStore

## CPFSR — تطبيق الهاتف المحمول

---

## المرحلة 1. تحضير المطوِّر

- [ ] تسجيل حساب في RuStore Console: https://console.rustore.ru
- [ ] تأكيد الحساب عبر Gosuslugi (للمطوّرين الروس).
- [ ] رفع المستندات:
  - [ ] جواز السفر أو وثيقة أخرى للشخص الطبيعي؛
  - [ ] الوثائق التأسيسية للشخص الاعتباري؛
  - [ ] شهادة INN.
- [ ] توقيع اتفاقية مطوِّر RuStore.

## المرحلة 2. تحضير التطبيق

### المصنوعات التقنية
- [ ] جُمِع ملف AAB (`app-release.aab`).
- [ ] حجم AAB ≤ 150 ميجابايت (حد RuStore).
- [ ] AAB موقَّع بـ release-keystore (لا debug-keystore).
- [ ] تمت مواءمة versionCode وversionName.
- [ ] minSdkVersion = 28 (Android 9).
- [ ] targetSdkVersion = 34 (Android 14).
- [ ] applicationId = `ru.rsfsr.app`.

### الامتثال
- [ ] يعمل التطبيق على الإصدار المستهدف الأدنى من Android (Android 9).
- [ ] جرى اختبار التطبيق على 5 أجهزة مختلفة على الأقل.
- [ ] معدل الأعطال < 0,5% وفق بيانات الاختبارات الداخلية.

### المحتوى
- [ ] أيقونة 512 × 512 PNG.
- [ ] Feature graphic 1024 × 500 PNG.
- [ ] 5 لقطات شاشة كحد أدنى (1080 × 1920 موصى به).
- [ ] فيديو ترويجي (اختياري لكن موصى به).
- [ ] وصف موجز (حتى 80 حرفاً).
- [ ] وصف كامل (حتى 4000 حرف).
- [ ] نص «ما الجديد» (حتى 500 حرف).
- [ ] الكلمات المفتاحية.

### المستندات القانونية
- [ ] نُشرَت سياسة الخصوصية على العنوان `https://dp-rsfsr.ru/privacy` (أو نطاق مؤقت).
- [ ] نُشرَت شروط الاستخدام على العنوان `https://dp-rsfsr.ru/terms`.
- [ ] أُكِّد عنوان موقع المطوِّر.
- [ ] بريد الدعم فعّال.

## المرحلة 3. تعبئة البطاقة في RuStore Console

### المعلومات الأساسية
- [ ] الاسم: «CPFSR».
- [ ] العنوان الفرعي: «الخدمة الروسية لتمويل التنمية الاجتماعية».
- [ ] الفئة: المالية.
- [ ] الفئة الفرعية: الاستثمارات.
- [ ] الفئة العمرية: 18+.
- [ ] السعر: مجاناً.

### الوصف
- [ ] عُبِّئ الوصف الموجز.
- [ ] عُبِّئ الوصف الكامل.
- [ ] حُدِّدت الكلمات المفتاحية.
- [ ] عُبِّئ «ما الجديد».

### الرسومات
- [ ] رُفِعَت الأيقونة.
- [ ] رُفِع Feature graphic.
- [ ] رُفِعَت 5–8 لقطات شاشة.
- [ ] رُفِع الفيديو الترويجي (إن وُجِد).

### المعلومات التقنية
- [ ] رُفِع ملف AAB.
- [ ] رُفِع Mapping (mapping.txt لـ ProGuard).
- [ ] التواقيع سليمة (لا يُستخدَم debug-keystore).

### تصنيف المحتوى
- [ ] عُبِّئ استبيان IARC / نظام الوسم الروسي.
- [ ] حُدِّدت الفئة العمرية 18+.

### التوطين
- [ ] اللغة الروسية — أساسية، عُبِّئت بالكامل.
- [ ] اللغة الإنجليزية — عُبِّئت.
- [ ] اللغة الفرنسية — عُبِّئت (اختياري).
- [ ] اللغة العربية — عُبِّئت (اختياري).

### المعلومات القانونية
- [ ] رابط إلى سياسة الخصوصية.
- [ ] رابط إلى شروط الاستخدام.
- [ ] بيانات الاتصال بالمطوِّر.
- [ ] أُشير إلى أن التطبيق لا يستخدم الإعلانات.
- [ ] أُشير إلى أن التطبيق لا يحتوي على مشتريات داخلية.

### المعلومات المالية (عند الانطباق)
- [ ] حُدِّد نوع النشاط المالي.
- [ ] رُفِعَت نُسَخ التراخيص (بنك روسيا — بعد الحصول).
- [ ] أُشير إلى الارتباط بالأنظمة الحكومية (Gosuslugi، EGRN).

## المرحلة 4. الأمن والتنظيم

### قبل النشر الإنتاجي
- [ ] الحصول على شهادة FSTEC الروسي (K1 / CII-1).
- [ ] الحصول على شهادة FSB الروسي (SZI).
- [ ] الحصول على إخطار Roskomnadzor بإدراج المشغِّل في سجل مشغّلي البيانات الشخصية.
- [ ] إدراج التطبيق في السجل الموحَّد للبرمجيات الروسية (قرار الحكومة رقم 1236).
- [ ] إدراج المشغِّل في سجل مشغّلي نظم المعلومات لدى بنك روسيا (النظام رقم 779-P).

### بديلاً: للوضع التجريبي
- [ ] الحصول على إذن بنك روسيا للتشغيل التجريبي.
- [ ] الإشارة الصريحة إلى وضع «التشغيل التجريبي» في التطبيق (في قسم About).
- [ ] تقييد الوصول إلى مجموعة المستخدمين التجريبيين عبر الجزء الخادم.

## المرحلة 5. الاختبار

### الاختبار الداخلي
- [ ] اجتياز اختبار ألفا (≥ 50 مستخدماً).
- [ ] اجتياز جميع السيناريوهات الحرجة (20 اختباراً).
- [ ] معالجة الأخطاء المكتشَفة.

### اختبار بيتا
- [ ] توزيع بناء بيتا عبر RuStore Beta أو قناة داخلية.
- [ ] جمع الملاحظات (≥ 100 مستخدم).
- [ ] جلسات بلا أعطال ≥ 99,8%.
- [ ] معالجة الأخطاء المكتشَفة.

## المرحلة 6. التقديم للمراجعة

- [ ] تعبئة جميع الحقول.
- [ ] رفع جميع المصنوعات.
- [ ] إرسال طلب المراجعة في RuStore Console.

### مدة المراجعة
**1–3 أيام عمل** (معتادة).

### النتائج المحتملة
- ✓ موافَق عليه — النشر في الكتالوج.
- ⚠ يحتاج إلى تعديل — ملاحظات المراقب → إجراء التصحيحات → إعادة التقديم.
- ✗ مرفوض — مخالفة حرجة → التحليل → المعالجة → إعادة التقديم.

## المرحلة 7. بعد النشر

### الرصد
- [ ] رصد التنزيلات في RuStore Console.
- [ ] رصد آراء المستخدمين.
- [ ] ربط تقارير الأعطال (AppMetrica / بديل محلي).
- [ ] رصد مؤشرات الأداء الرئيسية (DAU، MAU، retention).

### الدعم
- [ ] فريق الدعم جاهز للرد على الاستفسارات.
- [ ] مراقبة البريد support@dp-rsfsr.ru على مدار الساعة.
- [ ] روبوت المحادثة في التطبيق يعمل.

### التحديثات
- [ ] جدول التحديثات المنتظمة (1–2 مرة شهرياً).
- [ ] إجراء hot-fix للثغرات الحرجة.
- [ ] التوافق العكسي لـ API ≥ 12 شهراً.
