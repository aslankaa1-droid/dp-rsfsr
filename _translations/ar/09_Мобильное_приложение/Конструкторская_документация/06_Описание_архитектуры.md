# وصف بنية تطبيق الهاتف المحمول

## CPFSR — تطبيق الهاتف المحمول

**الرمز:** РСФСР.МП.06.ОА

---

## 1. النظرة العامة المعمارية

### 1.1. المنهج
بُنِي التطبيق وفق مبدأ **PWA-first** بالتغليف في أغلفة أصلية عبر **Capacitor 6**. وهذا يوفِّر قاعدة شيفرة موحَّدة لـ ويب وAndroid وiOS وWindows مع الحفاظ على الوصول إلى واجهات API الأصلية لكل منصة.

### 1.2. مزايا المنهج
- قاعدة شيفرة موحَّدة — توفير 60–70% من موارد التطوير.
- إمكانية إصدار المنصات باستقلالية.
- PWA كاملة للمستخدمين دون تثبيت.
- الوصول إلى واجهات API الأصلية عبر إضافات Capacitor.
- الامتثال لمتطلبات إحلال الواردات (دون الاعتماد على أطر عمل أجنبية متعددة المنصات).

## 3. هيكل المكوّنات

### 3.1. Presentation Layer
- **الشاشات** — 23 شاشة كأقسام HTML منفصلة بسمات data-screen فريدة.
- **المكوّنات** — عناصر UI قابلة لإعادة الاستخدام (البطاقات، الأزرار، النماذج، الرسوم البيانية).
- **السمات** — أربع سمات عبر CSS Custom Properties.
- **i18n** — دعم أربع لغات عبر قواميس JSON.

### 3.2. Application Layer
- **Router** — التنقّل عبر hash URL (`#screen-name`). دعم سجل المتصفح وزر «رجوع» نظام التشغيل.
- **State Manager** — تخزين مركزي لحالة المستخدم والأصول وDFA والسجل والإشعارات.
- **Theme Manager** — تبديل السمات مع الحفظ في LocalStorage.
- **i18n Manager** — توطين الواجهة مع التبديل التلقائي لاتجاه النص للعربية.
- **A11y Manager** — التحكم في حجم الخط وسمة التباين العالي والتركيز.

### 3.3. Domain Layer
كيانات الأعمال:
- `User` — مستخدم بمعرّف، اسم كامل، حالة KYC.
- `Asset` — أصل بنوع وخصائص وتقييم وحالة.
- `Valuation` — تقييم بمنهج وسعر ومدة صلاحية وتأمين.
- `SmartContract` — عقد ذكي بشروط وغرض وسعر وحالة.
- `CFA` — أصل مالي رقمي مرتبط بعقد ذكي ومدفوعات.
- `Transaction` — عملية في السجل (إصدار، تبادل، دفع).
- `Notification` — إشعار بفئة وحالة وإجراء.

### 3.4. Data Layer
- **Repositories** — تجريد للوصول إلى البيانات مع التبديل بين التخزين المحلي وواجهة API الشبكية.
- **API Client** — عميل REST API مُحدَّد الأنواع بمنطق retry وذاكرة تخزين مؤقت.
- **Local Storage** — LocalStorage للإعدادات، IndexedDB لذاكرة تخزين مؤقت للبيانات، المخزن المحمي لنظام التشغيل للأسرار.

### 3.5. Infrastructure
- **Service Worker** — تخزين مؤقت للموارد الساكنة واستجابات API؛ معالجة إشعارات push؛ العمل في وضع عدم الاتصال.
- **Crypto** — غلاف فوق WebCrypto API + إضافة أصلية لتشفير GOST (KriptoPro Mobile SDK).
- **Network** — Fetch API مع تحديث تلقائي للرموز؛ certificate pinning عبر الإضافات الأصلية.

## 6. أمن المعلومات

### 6.1. حماية التطبيق
- **Anti-tampering** — التحقق من سلامة الشيفرة عند التشغيل.
- **Anti-debug** — اكتشاف المُنقِّحات والمحاكيات.
- **كشف Jailbreak/Root** — رفض العمل على الأجهزة المخترَقة.
- **Certificate pinning** — تثبيت شهادات CA للحماية من MITM.
- **Anti-screenshot** — للشاشات الحرجة (التوقيع، العقد الذكي).
- **Obfuscation** — تشويش شيفرة JS عند البناء.

### 6.2. حماية البيانات
- **التشفير في الراحة** — جميع البيانات على الجهاز مشفَّرة بمفتاح من المخزن المحمي.
- **التشفير في النقل** — TLS بشهادات GOST.
- **عدم تخزين البيومترية** — فقط معرّفات للتحقق عبر EBS.
- **تدوير الرموز** — تحديث رموز الترخيص كل 15 دقيقة.

### 6.3. الامتثال للمعايير
- OWASP MASVS L2+.
- GOST R 57580.1 المستوى 4.
- FZ-152 (حماية البيانات الشخصية).
- FZ-187 (CII، الفئة 1).
- نظام بنك روسيا رقم 683-P.

## 7. الأداء

### 7.1. ميزانية الأداء

| المقياس | الميزانية |
|---|---|
| Cold start | ≤ 2 ثانية على 4G |
| Time to Interactive (TTI) | ≤ 3 ثوانٍ |
| Largest Contentful Paint | ≤ 1,5 ثانية |
| Cumulative Layout Shift | ≤ 0,05 |
| Bundle size (initial) | ≤ 200 كيلوبايت |
| إجمالي النقل | ≤ 1 ميجابايت عند التشغيل الأول |
| Memory footprint | ≤ 300 ميجابايت |
| البطارية في الخلفية | الحد الأدنى (no background tasks) |

### 7.2. التحسينات
- Lazy loading للشاشات والمكوّنات.
- Minification وtree-shaking لـ JS.
- ضغط الصور (WebP).
- التخزين المؤقت لاستجابات API عبر Service Worker.
- استخدام `requestIdleCallback` للمهام غير الحرجة.

## 8. إمكانية الوصول

- WCAG 2.2 المستوى AA (إلزامي)، AAA (مستهدف).
- دعم VoiceOver، TalkBack، Narrator.
- التكبير الديناميكي للخط.
- سمة عالية التباين لضعاف البصر.
- الحد الأدنى للمنطقة اللمسية 44 × 44 بكسل.
- سمات ARIA صحيحة لجميع العناصر التفاعلية.
- Skip links للتنقّل السريع.
