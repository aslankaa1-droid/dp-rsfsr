2. نموذج بيانات الخادم

النموذج الكامل للبيانات موصوف في الوثائق التقنية للمنصة (01_Техническая_документация/06_DLT_реестр_ЦФА.md). يستخدم العميل إسقاطات REST لنموذج الخادم.

الكيانات الرئيسية

  • Identity (المستخدم) — الارتباط بـ ESIA.
  • KycRecord — سجلات فحص KYC.
  • Asset — أصل مرتبط بـ EGRN/Rospatent/الإيداع.
  • Valuation — تقييم مع الشريك والمنهج والتأمين.
  • SmartContract — عقد ذكي بتواقيع جميع الأطراف.
  • CFAToken — رمز في سجل DLT بسجل كامل.
  • Escrow — سجل ضمان بنك روسيا.
  • Treasury — سجلات الخزانة الفيدرالية.
  • NationalProgramme — برنامج مشروع وطني.
  • InsurancePolicy — بوليصة تأمين.
  • EgrnRecord — سجل التقييد.
  • Notification — إشعار المستخدم.
  • AuditLog — سجل تدقيق غير قابل للتعديل لجميع العمليات.

3. العلاقات بين الكيانات

User ─< Asset ─< Valuation
        │
        └─< SmartContract ──> CFAToken ──> Escrow ──> Treasury
                                  │
                                  ├─> InsurancePolicy
                                  ├─> EgrnRecord
                                  └─< Transaction (مدفوعات)

User ─< Notification
User ─< AuditLog (read-only)

4. دورة حياة البيانات

4.1. Asset

DRAFT → PENDING_VALUATION → VALUED → CONTRACT_PENDING →
  ACTIVE → [RETURNED | REINVESTED | EXCHANGED]

4.2. CFA

PENDING_ISSUE → ISSUED → IN_CB_PORTFOLIO → ACTIVE →
  [MATURED → RETURNED | EXCHANGED_MENA | INSURED_CASE | REINVESTED]

6. النسخ الاحتياطي

تُكرَّر بيانات المستخدم تلقائياً في الجزء الخادم. ولا يخزّن الجهاز سوى ذاكرة مؤقتة لوضع عدم الاتصال. وعند تغيير الجهاز، تُستعاد جميع البيانات بعد الترخيص الأول على الجهاز الجديد.

7. حذف بيانات المستخدم

عند حذف حساب المستخدم وفق FZ-152: 1. البيانات على الجهاز — تُحذَف فوراً. 2. البيانات في قاعدة بيانات التشغيل — تُحذَف خلال 30 يوماً. 3. البيانات في سجل DLT (لـ DFA النشطة) — يجب رفع التقييد؛ ثم يبقى السجل بشكل أرشيفي مع علامة على حذف البيانات الشخصية للمالك، مع الحفاظ على التسلسل الزمني (متطلب التقارير الإشرافية). 4. النسخ الاحتياطية — تُحذَف وفق جدول الدوران (حتى 7 سنوات من الاحتفاظ وفق متطلبات بنك روسيا).

وصف تفصيلي لنموذج بيانات الخادم — في الوثائق التقنية للمنصة.
