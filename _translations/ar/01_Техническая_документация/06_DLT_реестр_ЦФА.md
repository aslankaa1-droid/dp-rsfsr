# سجل DLT لـ DFA في منصة CPFSR

يصف المستند بنية السجل الموزَّع لإصدار وتداول الأصول المالية الرقمية على أساس منصة DLT محلية وفق متطلبات FZ-259 ولوائح بنك روسيا رقم 779-P ورقم 780-P.

---

## 1. الأساس التقني

- منصة DLT الأساسية — نظام روسي معتمد من فئة «Masterchain» أو ما يكافئه (يُتَّفَق على المورِّد المحدَّد مع بنك روسيا في مرحلة التشغيل التجريبي).
- بنية السجل — **DLT permissioned** (سجل مغلق وموثوق بعدد عُقَد معروف).
- خوارزمية التوافق — Practical Byzantine Fault Tolerance (PBFT) أو نظيرها المحلي (RAFT مع ضمان النهائية).
- التشفير: GOST R 34.10-2012 (التوقيع)، GOST R 34.11-2012 (التجزئة).
- نهائية المعاملة — حتمية (لا احتمالية)، وهو ما يميزها عن سلاسل الكتل العامة.

## 2. تكوين عُقَد السجل

| نوع العقدة | العدد (الحد الأدنى) | المالك |
|---|---|---|
| عقدة المشغِّل (مع حق الكتابة) | 3 (توزّع جغرافي على 3 مراكز بيانات) | مشغِّل CPFSR |
| عقدة المنظِّم (قراءة فقط + إشراف) | 1 | بنك روسيا |
| عقدة الخزانة (قراءة + تأكيد العمليات المستهدفة) | 1 | الخزانة الفيدرالية |
| عقدة الشريك (المُقيِّمون، شركات التأمين) | 2+ (حسب عدد الشركاء المعتمَدين) | الشركاء |

جميع العُقَد متصلة بقنوات مؤمَّنة (ViPNet). تكوين العُقَد يُقَرّ من قِبَل بنك روسيا.

## 3. هيكل رمز DFA

كل رمز DFA هو سجل يحتوي على السمات التالية:

| السمة | النوع | الوصف |
|---|---|---|
| `cfa_id` | string | معرّف فريد `cfa-{hash6}-rsfsr-{year}-{seq}`. |
| `owner_id` | string | معرّف الراهن (مرتبط بـ ESIA). |
| `operator_id` | string | معرّف المشغِّل المُصدِر. |
| `cb_holder` | string | تأكيد الاستلام من قِبَل بنك روسيا. |
| `treasury_recipient` | string | معرّف المستفيد من الأموال في الخزانة الفيدرالية. |
| `asset_type` | enum | نوع الملكية (REAL_ESTATE، VEHICLE، IP، SECURITIES). |
| `asset_ref` | object | مرجع خارجي للأصل (رقم مساحي، VIN، معرّف Rospatent، ISIN). |
| `valuation` | money | قيمة التقييم، روبل. |
| `nominal` | money | القيمة الاسمية لـ DFA، روبل (افتراضياً ≤ 50% من التقييم). |
| `currency` | string | RUB (بما في ذلك الروبل الرقمي). |
| `term_months` | int | مدة الصلاحية بالأشهر. |
| `issue_date` | date | تاريخ الإصدار. |
| `maturity_date` | date | تاريخ الانتهاء. |
| `reward_rate` | decimal | معدل المكافأة للمالك، % سنوياً. |
| `target_program` | string | معرّف المشروع الوطني / البرنامج الفيدرالي. |
| `insurance` | object | بيانات التأمين (شركة، بوليصة، مبلغ). |
| `obremenenie_egrn` | string | معرّف قيد التقييد في EGRN. |
| `smart_contract_hash` | string | تجزئة GOST R 34.11-2012 لنص العقد الذكي. |
| `status` | enum | حالة DFA (انظر جدول الحالات أدناه). |
| `events` | array | سجل الأحداث (مع توقيعات UKEP). |
| `mena_history` | array | سجل عمليات التبادل مع الكائنات السابقة والجديدة. |
| `audit_signature` | string | UKEP للمشغِّل (GOST 34.10). |

## 4. حالات DFA

```
DRAFT
  ↓ (بعد التقييم والتأمين)
VALUED
  ↓ (بعد توقيع المالك على العقد الذكي)
CONTRACT_SIGNED
  ↓ (بعد تسجيل التقييد في EGRN)
ISSUED
  ↓ (بعد النقل إلى بنك روسيا)
TRANSFERRED_TO_CB
  ↓ (بعد تأكيد الإصدار)
ACTIVE
  ↓ ↓ ↓ ↓
  REINVESTED   — مدَّد المالك الرمز وأعاد إصداره.
  RETURNED     — انتهت المدة، رُفِع التقييد، أُعيدت الملكية.
  INSURED_CASE — وقعت حالة تأمين، صدر DFA تأميني.
  MENA_DONE    — جرى تبادل مع DFA لكائن مُنشَأ.
```

كل انتقال هو حدث في السجل، غير قابل للتعديل، بتوقيعات المشغّلين والمنظِّمين.

## 5. العقد الذكي لـ DFA

### 5.1. الشروط الجوهرية (وفق المادة 4 من FZ-259)

1. الأطراف.
2. معرّف الأصل وتقييمه.
3. القيمة الاسمية ومدة DFA.
4. شروط الاستخدام: الغرض المستهدف (مشروع وطني).
5. مبلغ وإجراءات سداد عمولات المشغِّل.
6. مبلغ وإجراءات مكافأة المالك.
7. شروط تأمين المخاطر.
8. شروط التبادل مع DFA للكائن المُنشَأ.
9. إجراءات توقيع محضر التسويات المتبادلة.
10. إجراءات رفع التقييد.

### 5.2. أحداث العقد الذكي
- `ON_ISSUE` — الإصدار.
- `ON_CB_RECEIPT` — استلام بنك روسيا.
- `ON_EMISSION_CONFIRMED` — تأكيد الإصدار مقابل DFA.
- `ON_QUARTERLY_REWARD` — دفع المكافأة الفصلية.
- `ON_INSURANCE_CASE` — حالة تأمين.
- `ON_MENA_REQUEST` — طلب التبادل.
- `ON_MENA_DONE` — اكتمال التبادل.
- `ON_RETURN` — إعادة الملكية إلى المالك.
- `ON_REINVESTMENT` — إعادة الاستثمار.

### 5.3. الطبيعة القانونية
- يُعَدّ العقد الذكي **عقداً ملزماً قانوناً** بمعنى المادتين 309–310 من القانون المدني والمادة 4 من FZ-259.
- التوقيع — UKEP وفق GOST R 34.10-2012 لمالك الأصل (عبر ESIA)، والمشغِّل، وبنك روسيا، وشركة التأمين.
- التنفيذ التقني — تلقائي (مدفوع بالأحداث).

## 6. حظر تداول DFA الحر (متطلب جوهري للمفهوم)

تُنفَّذ حظورات برمجية في العقد الذكي:

1. **نقل DFA** — فقط من المشغِّل إلى بنك روسيا ومن بنك روسيا إلى المشغِّل (للتبادل / الإعادة).
2. **تسييل DFA** — ليس لبنك روسيا الحق في تسييل DFA لمشاركي أسواق الأوراق المالية أو الأصول الرقمية.
3. **استخدام DFA خارج الغرض** — مستحيل برمجياً، وأحداث `ON_DISPOSAL_OUT_OF_SCOPE` تُرفَض بالتوافق.

هذه الحظورات مدمجة تقنياً في العقد الذكي وتتحكم بها عقدة المنظِّم في بنك روسيا.

## 7. الارتباط بدائرة إصدار بنك روسيا

- بعد حدث `ON_CB_RECEIPT` ترسل المنصة طلباً إلى منصة الروبل الرقمي لبنك روسيا لإصدار أموال مقابل القيمة الاسمية لـ DFA.
- يُسجَّل تأكيد الإصدار (حدث `ON_EMISSION_CONFIRMED`) في السجل.
- تُقيَّد الأموال في حساب ضمان خاص / محفظة رقمية لبنك روسيا بغرض مستهدف.
- التحويل إلى الخزانة الفيدرالية حدث مستقل `ON_TREASURY_FUNDING`.

## 8. التدقيق وعدم القابلية للتعديل

- جميع الأحداث موقَّعة بـ UKEP.
- كل كتلة في السجل تحتوي على تجزئة سابقتها (سلسلة الثقة).
- السجل الكامل لكل DFA متاح لبنك روسيا لحظياً.
- التدقيق الخارجي للسجل يُجرى سنوياً من قِبَل منظمة تدقيق معتمدة.

## 9. الأداء

| المعلَمة | القيمة |
|---|---|
| الإنتاجية | ≥ 200 عملية/ثانية |
| النهائية | ≤ 3 ثوانٍ (حتمية) |
| الكُمون لعمليات القراءة | ≤ 100 ميلي ثانية |
| حجم السجل | حتى 100 جيجابايت في السنة الأولى، ثم +200–400 جيجابايت/سنة |

## 10. التكرار وDR

- يُكرَّر السجل بصورة متزامنة على 3 مراكز بيانات للمشغِّل.
- نسخة احتياطية كاملة — يومياً، تُحفَظ 10 سنوات.
- اختبار الاستعادة — ربع سنوياً.

## 11. التقارير التنظيمية بشأن السجل

- تُرسَل لقطات منتظمة من السجل إلى بنك روسيا.
- عند طلب المنظِّم — تصدير كامل لسجل DFA محدد بالصيغة المعتمدة.
- التقارير إلى Rosfinmonitoring بشأن العمليات المشبوهة — تلقائياً.

## 12. إدارة العُقَد

- استبدال كامل لزوج مفاتيح العقدة — مرة كل 3 سنوات.
- إضافة عُقَد شركاء جدد — بقرار من مجلس المشغّلين بتأكيد بنك روسيا.
- إزالة عقدة (عند سحب ترخيص شريك) — وفق اللائحة، مع أرشفة سجلاتها التاريخية.
