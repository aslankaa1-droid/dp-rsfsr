التكاملات مع نظم المعلومات الحكومية

يصف المستند تكاملات منصة CPFSR مع نظم المعلومات الحكومية للاتحاد الروسي وفقاً لمتطلبات FZ-149، SMEV 3، FZ-152، FZ-187 ولوائح الجهات المالكة.

1. الجدول المجمَّع للتكاملات

المعرّف الطرف الغرض القناة البروتوكول التواتر الأمن
INT-01 Gosuslugi (ESIA) تعريف الأشخاص الطبيعيين والاعتباريين SMEV 3 + REST OpenID Connect / SAML 2.0 لحظي GOST TLS، UKEP
INT-02 النظام البيومتري الموحَّد (EBS) التعرّف البيومتري SMEV 3 REST عند الطلب KriptoPro TLS
INT-03 EGRN (Rosreestr) الحصول على معلومات العقارات، تسجيل التقييدات SMEV 3 + SOAP SOAP / REST عند الطلب GOST TLS، UKEP
INT-04 SMEV 3 تبادل الوثائق الإلكترونية بين الجهات SOAP SOAP 1.2 عند الطلب KriptoPro TLS
INT-05 GIS GMP المدفوعات والتسويات المتبادلة REST HTTPS دفعة يومية GOST TLS
INT-06 بنك روسيا — سجل مشغّلي نظم المعلومات تسجيل المشغِّل، التقارير قناة ViPNet مؤمَّنة XML / REST لائحي ViPNet، UKEP
INT-07 بنك روسيا — منصة الروبل الرقمي إصدار وتحويل الروبل الرقمي قناة مؤمَّنة بروتوكول داخلي لبنك روسيا لحظي ViPNet، GOST
INT-08 SPFS (بنك روسيا) الرسائل المالية بوابة SPFS تنسيق SPFS حسب الجدول ViPNet
INT-09 الخزانة الفيدرالية تحويل الأموال المستهدفة، التقارير SMEV 3 REST حسب الجدول GOST TLS، UKEP
INT-10 Rosfinmonitoring تقارير FZ-115 SMEV 3 XML / REST حسب الجدول GOST TLS، UKEP
INT-11 FNS الروسي التقارير الضريبية، التحقق من INN SMEV 3 REST يومي / عند الطلب GOST TLS
INT-12 FSTEC / FSB — اعتماد وسائل حماية المعلومات تفاعل لائحي تبادل وثائقي — — —
INT-13 NSPK (Mir) المدفوعات ببطاقة «Mir» قناة مؤمَّنة بروتوكول NSPK لحظي GOST TLS
INT-14 Mos.ru / Mos.reg التعريف الإقليمي (اختياري) SMEV 3 REST عند الطلب GOST TLS
INT-15 Rospatent تأكيد حقوق الملكية الفكرية SMEV 3 SOAP عند الطلب GOST TLS
INT-16 FNP (التوثيق) تسجيل رهن الممتلكات المنقولة API FNP REST عند الطلب GOST TLS، UKEP
INT-17 GIS ZhKKh معلومات عن العقارات (اختياري) SMEV 3 REST عند الطلب GOST TLS

2. اللوائح التفصيلية للتكاملات الرئيسية

2.1. ESIA (Gosuslugi)

  • المعيار: التوصيات المنهجية لتطبيق ESIA (وزارة التطوير الرقمي).
  • المصادقة: OpenID Connect 1.0.
  • مستوى الحساب: مؤكَّد (UZ-3).
  • السيناريوهات: التسجيل، الترخيص، الحصول على البيانات الشخصية بموافقة المستخدم، التحقق بين الجهات.
  • الخصوصيات: يلزم الاتصال عبر قناة مؤمَّنة؛ تُستوفى موافقة المستخدم على صفحة ESIA.

2.2. EBS (النظام البيومتري الموحَّد)

  • المعيار: نظام بنك روسيا رقم 482-P.
  • الغرض: التعرّف البيومتري لتقديم الأصول وتوقيع العقود عن بُعد.
  • نوع القياس البيومتري: الصوت + الوجه (متعدد الأنماط).
  • الخصوصيات: تعمل المنصة بوصفها مستهلكاً لـ EBS؛ يُحظَر نقل المتجهات البيومترية، يُسمَح فقط بالتحقق عبر المعرّف.

2.3. EGRN (Rosreestr)

  • المعيار: لائحة التفاعل الإلكتروني مع Rosreestr.
  • السيناريوهات:
    • الحصول على معلومات عن العقار حسب الرقم المساحي (للتقييم)؛
    • تقديم طلب التسجيل الحكومي للتقييد (الرهن)؛
    • تقديم طلب رفع التقييد بعد التبادل / انتهاء المدة.
  • الخصوصيات: جميع الطلبات تُوقَّع بـ UKEP لمشغِّل المنصة.

2.4. بنك روسيا — سجل مشغّلي نظم المعلومات

  • اللائحة: اللائحة رقم 779-P.
  • التقارير المنتظمة:
    • حجم DFA المُصدَرة (شهرياً)؛
    • سجل DFA النشطة (ربع سنوي)؛
    • المؤشرات المالية للمشغِّل (ربع سنوي)؛
    • حوادث أمن المعلومات (في غضون 24 ساعة من الاكتشاف).

2.5. بنك روسيا — منصة الروبل الرقمي

  • اللائحة: الفصل العاشر من FZ-86 (بعد رقم 340-FZ).
  • سيناريوهات المرحلة 1 (التشغيل التجريبي المخفّف 2026–2028): الروبل الرقمي مكمِّل تسوية (أحد خيارات الدفع غير النقدي لإصدار DFA)، دون ربط إصداري من بنك روسيا.
  • سيناريوهات المرحلة 2 (أفق 2029+، بعد اعتماد حزمة 4 تعديلات قانونية فيدرالية):
    • طلب إصدار الروبل الرقمي مقابل مجموعة DFA المرهونة؛
    • الحصول على تأكيد القيد على محفظة رقمية خاصة؛
    • بدء التحويل إلى محفظة الخزانة الفيدرالية الرقمية.

2.6. الخزانة الفيدرالية

  • الغرض: في المرحلة 1 — استلام أموال التمويل البِنيوي عبر مصنع VEB.RF (قرار الحكومة رقم 158) وقانون الميزانية؛ تأكيد الإنفاق المستهدف؛ تقرير تنفيذ المشروع. في المرحلة 2 (أفق 2029+) — استلام أموال الإصدار المضمون المستهدف من بنك روسيا في إطار التعديلات على قانون ميزانية الاتحاد الروسي.
  • البروتوكول: SMEV 3 + تقارير مؤمَّنة بالنماذج المعيارية للخزانة الفيدرالية.
  • الإشعارات: في كل مرحلة — إشعارات تلقائية في سجل DLT وسجل التدقيق.

2.7. Rosfinmonitoring

  • اللائحة: FZ-115، أمر Rosfinmonitoring رقم 366.
  • التقارير:
    • الرقابة الإلزامية على العمليات؛
    • العمليات المشبوهة (وفق نهج مستند إلى المخاطر)؛
    • معلومات المستفيدين الحقيقيين.
  • الخصوصيات: ترسل المنصة التقارير بصيغ XML المعتمدة للخدمة الإلكترونية الفيدرالية.

3. أمن التكاملات

  • جميع القنوات مُشفَّرة باستخدام GOST TLS / ViPNet.
  • جميع الطلبات — بمصادقة متبادلة mTLS.
  • جميع المعاملات الصادرة موقَّعة بـ UKEP للمشغِّل.
  • جميع الطلبات تُسجَّل في سجل التدقيق غير القابل للتعديل مع التتبّع.
  • جميع بوابات التكامل تقع في قطاع DMZ بـ WAF مخصَّص.

4. إدارة القاعدة التعاقدية للتكاملات

الطرف نوع العقد المدة
بنك روسيا اتفاقية تفاعل معلوماتي غير محدد، مع مراجعة سنوية
الخزانة الفيدرالية اتفاقية تفاعل معلوماتي غير محدد
وزارة التطوير الرقمي (SMEV 3، ESIA، EBS) لائحة الاتصال 5 سنوات
Rosreestr اتفاقية الاتصال 5 سنوات
Rosfinmonitoring لائحة الاتصال 5 سنوات
FNS اتفاقية SMEV 5 سنوات
NSPK اتفاقية تقنية 3 سنوات

5. تطوّر التكاملات

  • المرحلة 1 (2025–2026): المجموعة الأساسية INT-01–INT-11.
  • المرحلة 2 (2026–2027): إضافة INT-12–INT-17 وسيناريوهات موسَّعة للروبل الرقمي.
  • المرحلة 3 (2027+): تكاملات بين الدول (BRICS+، الاتحاد الاقتصادي الأوراسي).

6. إدارة حوادث التكاملات

  • رصد توافر كل تكامل على مدار الساعة.
  • تنبيهات عند تجاوز العتبات (الكُمون، معدل الأخطاء).
  • مراجعات منتظمة مع فِرَق الجهات المالكة.
  • قنوات احتياطية للتكاملات الحرجة (بنك روسيا، الخزانة الفيدرالية).
