# المخططات المعمارية لـ CPFSR

يحتوي المستند على أوصاف نصية لمخططات C4 للمنصة (Mermaid + وصف منظَّم). تُعدّ المخططات المرئية بواسطة مصمِّم ضمن الحزمة المرافقة.

---

## 1. C4 — المستوى 1: System Context

```mermaid
graph TB
    subgraph "البيئة"
        OWN["الراهن<br/>فرد / شخص اعتباري / مؤسسة حكومية / منظمة غير ربحية"]
        VAL["منظمة تقييم<br/>(شريك المنصة)"]
        INS["منظمة تأمين<br/>(شريك المنصة)"]
        CB["البنك المركزي<br/>للاتحاد الروسي"]
        FK["الخزانة الفيدرالية"]
        FR["Rosfinmonitoring"]
        ROSR["Rosreestr (EGRN)"]
        GOSUS["Gosuslugi (ESIA / EBS)"]
        SMEV["SMEV 3"]
        NPRA["عميل حكومي<br/>(وزارات، جهات)"]
        FSB["FSTEC / FSB / NCCCI<br/>(إشراف CII)"]
    end

    RSFSR["**CPFSR**<br/>منصة المشغِّل<br/>لإصدار DFA"]

    OWN -->|تقديم الأصل| RSFSR
    RSFSR -->|التقييم| VAL
    VAL -->|النتيجة| RSFSR
    RSFSR -->|التأمين| INS
    RSFSR <-->|DFA / السجل| CB
    CB <-->|تمويل مستهدف / ضمان| FK
    FK -->|التمويل| NPRA
    RSFSR -->|تقارير FZ-115| FR
    RSFSR <-->|التقييدات| ROSR
    RSFSR <-->|التعريف| GOSUS
    RSFSR -->|التبادل بين الجهات| SMEV
    RSFSR <-->|إشراف CII| FSB
```

### الوصف
CPFSR هي المنصة المركزية التي تربط أربعة أنواع من أصحاب المصلحة:
- **مالكو الأصول** — الأشخاص الطبيعيون والاعتباريون، المؤسسات الحكومية، المنظمات غير الربحية، المؤسسات.
- **الشركاء** — منظمات التقييم والتأمين المعتمدة من المنصة.
- **الجهات التنظيمية** — بنك روسيا، Rosfinmonitoring، FSTEC/FSB.
- **الجهات الحكومية** — الخزانة الفيدرالية والعملاء الحكوميون للمشاريع الوطنية.

### الوصف
تتكوّن المنصة من ثلاث طبقات رئيسية:
1. **القنوات** — تطبيقات الويب والهاتف للراهن، إضافة إلى حسابات الشركاء.
2. **Edge + الخدمات** — WAF، API Gateway، الخدمات المصغّرة.
3. **البيانات + التكاملات** — قواعد البيانات، التخزين الكائني، سجل DLT، التخزين المؤقت، وسيط الرسائل، نظم المعلومات الخارجية.

## 6. Sequence — إصدار DFA (E2E)

```mermaid
sequenceDiagram
    participant U as الراهن
    participant APP as Mobile App
    participant API as API Gateway
    participant KYC as KYC Service
    participant CAT as Catalog
    participant ROSR as EGRN
    participant VAL as Valuation
    participant SC as Smart Contract Engine
    participant CFA as CFA Issuer
    participant DLT as DLT Registry
    participant CB as بنك روسيا
    participant FK as الخزانة الفيدرالية

    U->>APP: التسجيل
    APP->>API: AuthN (ESIA)
    API->>KYC: تحقق FZ-115
    KYC-->>API: OK
    U->>APP: تقديم الأصل
    APP->>API: createAsset
    API->>CAT: createAsset
    CAT->>ROSR: getAssetInfo
    ROSR-->>CAT: بيانات EGRN
    CAT-->>API: assetId
    U->>APP: طلب التقييم
    APP->>API: requestValuation
    API->>VAL: createValuationOrder
    VAL-->>API: pending
    Note over VAL: المُقيِّم الشريك<br/>ينفِّذ التقييم
    VAL-->>API: valuationDone
    U->>APP: توقيع العقد الذكي
    APP->>API: signContract (UKEP)
    API->>SC: build & sign
    SC->>CFA: requestIssue
    CFA->>DLT: mint(cfaId)
    DLT-->>CFA: OK
    CFA->>CB: transferToCB
    CB-->>CFA: receipt
    CB->>FK: emission
    FK-->>CB: confirmed
    CB-->>CFA: ACTIVE
    CFA-->>API: cfaId ACTIVE
    API-->>APP: success
    APP-->>U: تم إصدار DFA
```

## 7. Sequence — تبادل DFA

```mermaid
sequenceDiagram
    participant FK as الخزانة الفيدرالية
    participant NPRA as عميل المشروع الوطني
    participant ROSR as EGRN
    participant CFA as CFA Issuer
    participant DLT as DLT Registry
    participant CB as بنك روسيا
    participant OWN as الراهن

    NPRA->>FK: محضر إدخال الكائن للتشغيل
    FK->>ROSR: تسجيل حق الملكية
    ROSR-->>FK: شهادة
    FK->>CFA: طلب إصدار DFA جديد
    CFA->>DLT: mint(cfaIdNew, تقييم جديد)
    DLT-->>CFA: OK
    CFA->>CB: طلب تبادل DFA المالك القديم بالجديد
    CB-->>CFA: تم التأكيد
    DLT->>DLT: status MENA_DONE
    CFA->>ROSR: رفع التقييد عن القديم
    ROSR-->>CFA: OK
    CFA->>OWN: المكافأة
    OWN-->>CFA: ACK
```
