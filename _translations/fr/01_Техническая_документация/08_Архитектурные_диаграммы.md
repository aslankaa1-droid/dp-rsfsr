# Diagrammes d'architecture de CPFSR

Le document contient des descriptions textuelles des diagrammes C4 de la plateforme (Mermaid + description structurée). Les diagrammes visuels seront élaborés par un designer dans le cadre du pack d'accompagnement.

---

## 1. C4 — Niveau 1 : System Context

```mermaid
graph TB
    subgraph "Environnement"
        OWN["Débiteur sur gage<br/>particulier / personne morale / entreprise d'État / OBNL"]
        VAL["Organisme d'évaluation<br/>(partenaire de la plateforme)"]
        INS["Organisme d'assurance<br/>(partenaire de la plateforme)"]
        CB["Banque centrale<br/>de la Fédération de Russie"]
        FK["Trésor fédéral"]
        FR["Rosfinmonitoring"]
        ROSR["Rosreestr (EGRN)"]
        GOSUS["Gosuslugi (ESIA / EBS)"]
        SMEV["SMEV 3"]
        NPRA["Client public<br/>(ministères, administrations)"]
        FSB["FSTEC / FSB / NCCCI<br/>(supervision IIC)"]
    end

    RSFSR["**CPFSR**<br/>Plateforme opérateur<br/>d'émission d'AFN"]

    OWN -->|soumission d'actif| RSFSR
    RSFSR -->|évaluation| VAL
    VAL -->|résultat| RSFSR
    RSFSR -->|assurance| INS
    RSFSR <-->|AFN / registre| CB
    CB <-->|Financement ciblé / escrow| FK
    FK -->|financement| NPRA
    RSFSR -->|reporting FZ-115| FR
    RSFSR <-->|charges| ROSR
    RSFSR <-->|identification| GOSUS
    RSFSR -->|échange interministériel| SMEV
    RSFSR <-->|supervision IIC| FSB
```

### Description
CPFSR est la plateforme centrale qui relie quatre types de parties prenantes :
- **Propriétaires d'actifs** — personnes physiques et morales, entreprises d'État, OBNL, fondations.
- **Partenaires** — organismes d'évaluation et d'assurance accrédités par la plateforme.
- **Régulateurs** — Banque de Russie, Rosfinmonitoring, FSTEC/FSB.
- **Organes étatiques** — Trésor fédéral et clients étatiques de projets nationaux.

## 5. Deployment Diagram (simplifié)

```mermaid
graph TB

    subgraph "Centre de données 2 (DF de l'Oural)"

    end
    subgraph "Centre de données 3 (DF de Sibérie)"
        K8S3["k8s cluster #3"]
        PG3["Postgres Pro #3"]
        DLT3["DLT Node #3"]
        HSM3["HSM #3"]
    end
    subgraph "Centre DR (de secours)"
        K8S4["k8s cluster (cold)"]
        BACK["Backup storage"]
    end

    K8S1 <--> K8S2
    K8S2 <--> K8S3
    K8S1 <--> K8S3
    PG1 <-->|sync repl| PG2
    PG2 <-->|sync repl| PG3
    DLT1 <-->|consensus| DLT2
    DLT2 <-->|consensus| DLT3
    DLT1 <-->|consensus| DLT3
    BACK <-- K8S1
    BACK <-- K8S2
    BACK <-- K8S3
```

### Description
- Topologie active-active-active sur 3 centres de données situés dans différents districts fédéraux.
- Réplication synchrone des BD.
- Consensus distribué des nœuds DLT.
- Centre de données « froid » de secours pour les scénarios DR.

## 6. Sequence — émission d'AFN (E2E)

```mermaid
sequenceDiagram
    participant U as Débiteur sur gage
    participant APP as Mobile App
    participant API as API Gateway
    participant KYC as KYC Service
    participant CAT as Catalog
    participant ROSR as EGRN
    participant VAL as Valuation
    participant SC as Smart Contract Engine
    participant CFA as CFA Issuer
    participant DLT as DLT Registry
    participant CB as BdR
    participant FK as Trésor fédéral

    U->>APP: Inscription
    APP->>API: AuthN (ESIA)
    API->>KYC: vérification FZ-115
    KYC-->>API: OK
    U->>APP: Soumission d'actif
    APP->>API: createAsset
    API->>CAT: createAsset
    CAT->>ROSR: getAssetInfo
    ROSR-->>CAT: données EGRN
    CAT-->>API: assetId
    U->>APP: Demander l'évaluation
    APP->>API: requestValuation
    API->>VAL: createValuationOrder
    VAL-->>API: pending
    Note over VAL: L'évaluateur partenaire<br/>réalise l'évaluation
    VAL-->>API: valuationDone
    U->>APP: Signer le contrat intelligent
    APP->>API: signContract (UKEP)
    API->>SC: build & sign
    SC->>CFA: requestIssue
    CFA->>DLT: mint(cfaId)
    DLT-->>CFA: OK
    CFA->>CB: transferToCB
    CB-->>CFA: receipt
    CB->>FK: emission
    FK-->>CB: confirmed
    CB-->>CFA: ACTIVE
    CFA-->>API: cfaId ACTIVE
    API-->>APP: success
    APP-->>U: AFN émis
```

## 7. Sequence — échange d'AFN

```mermaid
sequenceDiagram
    participant FK as Trésor fédéral
    participant NPRA as Client du projet national
    participant ROSR as EGRN
    participant CFA as CFA Issuer
    participant DLT as DLT Registry
    participant CB as BdR
    participant OWN as Débiteur sur gage

    NPRA->>FK: acte de mise en exploitation
    FK->>ROSR: enregistrement du droit de propriété
    ROSR-->>FK: certificat
    FK->>CFA: demande d'émission d'un nouvel AFN
    CFA->>DLT: mint(cfaIdNew, nouvelle évaluation)
    DLT-->>CFA: OK
    CFA->>CB: demande d'échange de l'ancien AFN du propriétaire contre le nouveau
    CB-->>CFA: confirmé
    DLT->>DLT: status MENA_DONE
    CFA->>ROSR: lever la charge sur l'ancien
    ROSR-->>CFA: OK
    CFA->>OWN: rémunération
    OWN-->>CFA: ACK
```
