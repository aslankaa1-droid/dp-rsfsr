Intégrations avec les systèmes d’information d’État

Le document décrit les intégrations de CPFSR avec les systèmes d’information d’État de la Fédération de Russie conformément aux exigences de FZ-149, SMEV 3, FZ-152, FZ-187 et aux règlements des administrations propriétaires.

2. Règlements détaillés des intégrations clés

2.1. ESIA (Gosuslugi)

  • Norme : Recommandations méthodologiques sur l’application d’ESIA (Ministère du Développement numérique).
  • Authentification : OpenID Connect 1.0.
  • Niveau du compte : Confirmé (UZ-3).
  • Scénarios : inscription, autorisation, obtention de données personnelles sur consentement, contrôle interministériel.
  • Particularités : connexion via un canal sécurisé requise ; le consentement est recueilli sur la page d’ESIA.

2.2. EBS (Système biométrique unifié)

  • Norme : Règlement de la Banque de Russie N° 482-P.
  • Objet : identification biométrique pour la soumission à distance d’actifs et la signature de contrats.
  • Type de biométrie : voix + visage (multimodale).
  • Particularités : la plateforme intervient en tant que consommatrice d’EBS ; la transmission des vecteurs biométriques est interdite, seule la vérification par identifiant est autorisée.

2.3. EGRN (Rosreestr)

  • Norme : Règlement d’interaction électronique avec Rosreestr.
  • Scénarios :
    • obtention d’informations sur un bien immobilier par numéro cadastral (pour l’évaluation) ;
    • dépôt d’une demande d’enregistrement étatique d’une charge (gage) ;
    • dépôt d’une demande de levée de la charge après échange / expiration du terme.
  • Particularités : toutes les demandes sont signées avec l’UKEP de l’opérateur de la plateforme.

2.4. Banque de Russie — registre des opérateurs SI

  • Règlement : Règlement N° 779-P.
  • Rapports réguliers :
    • volume d’AFN émis (mensuel) ;
    • registre des AFN actifs (trimestriel) ;
    • indicateurs financiers de l’opérateur (trimestriel) ;
    • incidents SI (dans les 24 h suivant la détection).

2.5. Banque de Russie — plateforme du rouble numérique

  • Règlement : Chapitre X de FZ-86 (après N° 340-FZ).
  • Scénarios de l’étape 1 (pilote-light 2026–2028) : le rouble numérique est un complément de règlement (l’une des options de paiement non monétaire pour l’émission d’AFN), sans lien d’émission de la part de la Banque de Russie.
  • Scénarios de l’étape 2 (horizon 2029+, après l’adoption du paquet des 4 amendements à la loi fédérale) :
    • demande d’émission de rouble numérique sur le pool d’AFN gagés ;
    • obtention de la confirmation de crédit sur un portefeuille numérique spécial ;
    • initiation d’un transfert vers le portefeuille numérique du Trésor fédéral.

2.6. Trésor fédéral

  • Objet : à l’étape 1 — réception des fonds de financement d’infrastructure via l’Usine de VEB.RF (décret du Gouvernement N° 158) et le Code budgétaire ; confirmation des dépenses ciblées ; rapport sur la mise en œuvre du projet. À l’étape 2 (horizon 2029+) — réception des fonds d’émission garantie ciblée de la Banque de Russie dans le cadre des amendements au Code budgétaire de la FR.
  • Protocole : SMEV 3 + rapports sécurisés sous les formes standardisées du FK.
  • Notifications : à chaque étape — notifications automatiques dans le registre DLT et le journal d’audit.

2.7. Rosfinmonitoring

  • Règlement : FZ-115, arrêté de Rosfinmonitoring N° 366.
  • Reporting :
    • contrôle obligatoire des opérations ;
    • opérations suspectes (selon une approche par les risques) ;
    • informations sur les bénéficiaires effectifs.
  • Particularités : la plateforme envoie les rapports dans les formats XML établis du FES.

3. Sécurité des intégrations

  • Tous les canaux sont chiffrés au moyen de GOST TLS / ViPNet.
  • Toutes les requêtes — avec authentification mutuelle mTLS.
  • Toutes les transactions sortantes sont signées par l’UKEP de l’opérateur.
  • Toutes les requêtes sont consignées dans le journal d’audit immuable avec traçage.
  • Toutes les passerelles d’intégration sont placées dans le segment DMZ avec un WAF dédié.

5. Évolution des intégrations

  • Phase 1 (2025–2026) : ensemble de base INT-01–INT-11.
  • Phase 2 (2026–2027) : ajout d’INT-12–INT-17 et de scénarios étendus du rouble numérique.
  • Phase 3 (2027+) : intégrations interétatiques (BRICS+, UEEA).

6. Gestion des incidents d’intégrations

  • Surveillance 24×7 de la disponibilité de chaque intégration.
  • Alertes en cas de dépassement de seuils (latence, taux d’erreurs).
  • Revues régulières avec les équipes des administrations propriétaires.
  • Canaux de secours pour les intégrations critiques (Banque de Russie, Trésor fédéral).
