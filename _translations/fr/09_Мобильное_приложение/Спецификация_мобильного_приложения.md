# Spécification de l'application mobile CPFSR

**Plateformes :** iOS / Android / Windows.
**Public cible :** débiteurs sur gage — personnes physiques et morales, représentants d'entreprises d'État, entrepreneurs individuels, OBNL et fondations.
**Objectifs clés :** assurer la gestion à distance sécurisée des actifs, l'émission et le service des AFN, la perception de la rémunération.

---

## 1. Pile technologique

### 1.1. Mise en œuvre multiplateforme
- **Plateforme de base :** PWA (HTML/CSS/JS + TypeScript), Service Worker pour le mode hors ligne et la mise à jour en arrière-plan, manifest avec shortcuts et share_target.
- **Wrapper iOS / Android :** Capacitor v6+ (wrappers iOS/Android natifs autour du noyau PWA) ; code base unique, accès natif à la biométrie, aux SZI GOST, à Secure Enclave/Strongbox, au certificate pinning, au push, au share_target.
- **Windows desktop :** PWA via Edge WebView2 (installation séparée via Microsoft Store / MSIX) ou edge_side_panel.
- **Logique métier commune :** TypeScript + modules WebAssembly pour les fonctions critiques (cryptographie, signature).
- **Système de design :** basé sur les tokens de marque du projet (`_shared/brand.css`), avec des overrides natifs pour iOS (SF Pro) et Android (Roboto).

### 1.2. Architecture de l'application client
- Pattern : PWA Shell + écrans en chargement paresseux.
- State management : un state-store interne dans `app.js` (vanilla JS) ou un adaptateur Zustand en cas de croissance de la complexité.
- Navigation : back-stack PWA via History API + drawer-aside en modes pleine taille.
- Client API : auto-génération à partir d'OpenAPI 3.1.

### 1.3. Stockage de données sur l'appareil
- Données sensibles : iOS Keychain / Android Keystore / Windows Credential Locker — uniquement des jetons chiffrés.
- Les données biométriques ne sont pas stockées localement — seuls les identifiants pour la vérification via EBS.
- Cache local — SQLCipher (SQLite chiffré).

### 1.4. Cryptographie
- UKEP sur l'appareil mobile — via KriptoPro EP (SDK mobile).
- Hachage — GOST R 34.11-2012.
- Canaux TLS — au moyen de certificats GOST.

## 2. Sécurité

### 2.1. Identification et authentification
1. **Autorisation primaire** — via ESIA (OpenID Connect) avec un compte confirmé.
2. **Facteurs supplémentaires :**
   - Biométrie de l'appareil (TouchID / FaceID / Windows Hello) pour l'accès à l'application.
   - Biométrie via EBS (règlement de la Banque de Russie N° 482-P) pour la signature des opérations critiques.
   - OTP (SMS / push) pour les opérations critiques (signature du contrat intelligent).

### 2.2. Protection de l'application
- **Anti-tampering :** Mobile Application Shielding (MAS) — solutions de classe nationale.
- **Jailbreak / Root Detection** — refus de fonctionnement sur les appareils compromis.
- **Anti-debug** — obfuscation, protection contre les injections d'exécution.
- **Anti-screenshot** pour les écrans critiques (contrat intelligent, OTP).
- **Certificate pinning** sur les canaux TLS.

### 2.3. Protection anti-fraude
- Analytique comportementale (biométrie comportementale).
- Device fingerprinting.
- Géolocalisation et patterns temporels.

### 2.4. Stockage des secrets
- Uniquement en TEE (Trusted Execution Environment) / Secure Enclave.
- Pas de clés en mémoire partagée de l'application.

### 2.5. Conformité
- FZ-152 (protection des données personnelles).
- FZ-187 (IIC) — l'application fait partie d'un objet IIC significatif de catégorie 1.
- Instruction de la Banque de Russie N° 6635-U (incidents SI).
- Standard OWASP MASVS Level 2+.
- Conformité WCAG 2.2 AA (accessibilité).

## 3. Modules fonctionnels

### 3.1. Inscription et identification
- Autorisation via ESIA.
- Vérification biométrique via EBS.
- Liaison de l'appareil.
- Consentements au traitement des données personnelles.

### 3.2. Espace personnel
- Tableau de bord : solde global des actifs en activité, revenu courant, portefeuille AFN.
- Liste des actifs avec filtres et recherche.
- Chaque actif : caractéristiques, historique, AFN courant.

### 3.3. Soumission d'un actif en garantie
- Sélection du type de bien.
- Chargement des données depuis les registres d'État (EGRN, Rospatent, dépositaires).
- Attachement de documents.
- Demande d'évaluation.

### 3.4. Évaluation
- Consultation du statut de la demande.
- Résultat de l'évaluation avec possibilité d'appel.
- Informations d'assurance.

### 3.5. Contrat intelligent
- Concepteur de conditions.
- Consultation du texte intégral du contrat.
- Signature électronique avec UKEP (certificat mobile).

### 3.6. Gestion des AFN
- Liste des AFN actifs.
- État de chaque AFN dans le registre DLT.
- Événements attendus (échange, sinistres, échéance).

### 3.7. Revenus
- Historique des paiements.
- Synthèse par trimestre et par an.
- Téléchargement de l'attestation 2-NDFL.

### 3.8. Notifications
- Push-notifications pour les événements critiques.
- Notifications via Gosuslugi (pour consentements, modifications).

### 3.9. Profil
- Gestion des coordonnées.
- Gestion de la sécurité (biométrie, canal OTP).
- Gestion des consentements.
- Coordonnées de versement.

### 3.10. Aide et support
- FAQ.
- Chat-bot.
- Contact avec l'opérateur de support.

## 4. Exigences non fonctionnelles

| Catégorie | Valeur cible |
|---|---|
| Temps du premier écran (cold start) | ≤ 2 sec en 4G |
| Temps de réponse UI | ≤ 100 ms |
| Taille de l'application (install) | ≤ 80 Mo |
| Prise en charge iOS | iOS 15+ |
| Prise en charge Android | Android 9 (API 28)+ |
| Prise en charge Windows | Windows 10 (1809)+ / Windows 11 |
| Langues d'interface | russe (principal), anglais (secondaire) |
| Thème sombre | Pris en charge |
| Accessibilité | Prise en charge de VoiceOver / TalkBack / Narrator |
| Mode hors ligne | Consultation des données mises en cache |

## 5. Plan de release

### 5.1. iOS / Android
- Release bêta via TestFlight / Google Play Internal Testing — mois 10.
- Tests fermés — mois 12.
- Lancement public — mois 18 (simultanément avec la plateforme).
- Publication dans RuStore (Ministère du Développement numérique).

### 5.2. Windows / Desktop (Capacitor v6+ pour desktop / PWA installable)
- Release bêta — mois 14 (PWA via Edge / Chrome « Install app » ; bundle Capacitor Windows complet — mois 16).
- Lancement public — mois 18.
- Publication dans Microsoft Store (via le paquet MSIX généré par Capacitor) + canaux alternatifs (web-distribution PWA).
- Avantage d'une code base PWA unique : les mêmes sources que pour iOS/Android — pas de pile .NET MAUI parallèle.

### 5.3. Mises à jour
- Auto-update via les stores standards.
- Mises à jour forcées pour les vulnérabilités critiques via un mécanisme in-app.

## 6. Tests

### 6.1. Types de tests
- Tests unitaires (Vitest + @testing-library/dom pour la logique PWA, JUnit/XCTest pour les plugins Capacitor natifs) — couverture ≥ 80 %.
- Tests d'intégration (Playwright Component Testing) — scénarios critiques.
- Tests E2E (Playwright cross-browser + Capacitor + Appium pour les wrappers natifs) — flux utilisateurs principaux pour iOS / Android / desktop.
- Tests de performance (Lighthouse CI ≥ 90 sur les 4 métriques PWA) sur différents appareils.
- Tests de sécurité (OWASP MASVS L2+ FT-021..025, ZAP / Burp, OWASP Mobile Top 10, scanners certifiés FSTEC).
- Tests d'accessibilité (axe-core, WCAG 2.2 AA, lecteurs d'écran iOS VoiceOver / Android TalkBack).

### 6.2. Parc d'appareils
- iOS : iPhone 13 / 14 / 15 / 16 ; iPad 9-gen / Pro.
- Android : Samsung Galaxy S22+, Xiaomi Redmi Note 12, Realme C55 (segment budget).
- Windows : Surface Pro 9, postes de travail standards.

### 6.3. CI/CD
- Build dans un environnement sécurisé avec signature UKEP.
- Tests automatiques à chaque merge.
- Branches de release + tags.
