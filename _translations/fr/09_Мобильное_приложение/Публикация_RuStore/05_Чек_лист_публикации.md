# Check-list de publication dans RuStore

## CPFSR — application mobile

---

## Étape 1. Préparation du développeur

- [ ] Compte enregistré dans RuStore Console : https://console.rustore.ru
- [ ] Compte confirmé via Gosuslugi (pour les développeurs russes).
- [ ] Documents chargés :
  - [ ] passeport ou autre document pour une personne physique ;
  - [ ] documents constitutifs pour une personne morale ;
  - [ ] certificat INN.
- [ ] Accord développeur RuStore signé.

## Étape 2. Préparation de l'application

### Artefacts techniques
- [ ] Fichier AAB compilé (`app-release.aab`).
- [ ] Taille de l'AAB ≤ 150 Mo (limite RuStore).
- [ ] AAB signé avec une release keystore (pas une debug keystore).
- [ ] versionCode et versionName alignés.
- [ ] minSdkVersion = 28 (Android 9).
- [ ] targetSdkVersion = 34 (Android 14).
- [ ] applicationId = `ru.rsfsr.app`.

### Conformité
- [ ] L'application fonctionne sur la version Android cible minimale (Android 9).
- [ ] L'application a été testée sur au moins 5 appareils différents.
- [ ] Crash-rate < 0,5 % selon les données des tests internes.

### Contenu
- [ ] Icône 512 × 512 PNG.
- [ ] Feature graphic 1024 × 500 PNG.
- [ ] Minimum 5 captures d'écran (1080 × 1920 recommandé).
- [ ] Vidéo promo (optionnel, mais recommandé).
- [ ] Description brève (jusqu'à 80 caractères).
- [ ] Description complète (jusqu'à 4 000 caractères).
- [ ] Texte « Quoi de neuf » (jusqu'à 500 caractères).
- [ ] Mots-clés.

### Documents juridiques
- [ ] Politique de confidentialité publiée à l'adresse `https://dp-rsfsr.ru/privacy` (ou un domaine temporaire).
- [ ] Conditions d'utilisation publiées à l'adresse `https://dp-rsfsr.ru/terms`.
- [ ] Adresse du site du développeur confirmée.
- [ ] E-mail de contact du support actif.

## Étape 3. Remplissage de la fiche dans RuStore Console

### Informations principales
- [ ] Nom : « CPFSR ».
- [ ] Sous-titre : « Service Russe de Financement du Développement Social ».
- [ ] Catégorie : Finance.
- [ ] Sous-catégorie : Investissement.
- [ ] Catégorie d'âge : 18+.
- [ ] Prix : Gratuit.

### Description
- [ ] Description brève remplie.
- [ ] Description complète remplie.
- [ ] Mots-clés indiqués.
- [ ] « Quoi de neuf » rempli.

### Graphique
- [ ] Icône téléchargée.
- [ ] Feature graphic téléchargé.
- [ ] 5–8 captures d'écran téléchargées.
- [ ] Vidéo promo téléchargée (le cas échéant).

### Informations techniques
- [ ] Fichier AAB téléchargé.
- [ ] Mapping (mapping.txt pour ProGuard) téléchargé.
- [ ] Signatures en ordre (debug keystore non utilisé).

### Classification du contenu
- [ ] Questionnaire IARC / système de marquage russe rempli.
- [ ] Catégorie d'âge 18+ définie.

### Localisation
- [ ] Langue russe — principale, remplie intégralement.
- [ ] Langue anglaise — remplie.
- [ ] Langue française — remplie (optionnel).
- [ ] Langue arabe — remplie (optionnel).

### Informations juridiques
- [ ] Lien vers la Politique de confidentialité.
- [ ] Lien vers les Conditions d'utilisation.
- [ ] Coordonnées du développeur.
- [ ] Indiqué que l'application n'utilise pas de publicité.
- [ ] Indiqué que l'application n'a pas d'achats internes.

### Informations financières (le cas échéant)
- [ ] Type d'activité financière indiqué.
- [ ] Copies des licences téléchargées (Banque de Russie — après obtention).
- [ ] Lien avec les systèmes d'État (Gosuslugi, EGRN) indiqué.

## Étape 4. Sécurité et réglementation

### Avant publication en production
- [ ] Certificat FSTEC de Russie obtenu (K1 / IIC-1).
- [ ] Certificat FSB de Russie obtenu (SZI).
- [ ] Notification Roskomnadzor d'inscription au registre des opérateurs de DCP obtenue.
- [ ] Application inscrite au Registre unique des logiciels russes (décret du Gouvernement N° 1236).
- [ ] Opérateur inscrit au registre des opérateurs SI de la Banque de Russie (Règlement N° 779-P).

### Alternativement : pour le mode pilote
- [ ] Autorisation de la Banque de Russie pour l'exploitation pilote obtenue.
- [ ] L'application indique explicitement le statut « Mode pilote » (dans la section À propos).
- [ ] L'accès est limité au pool d'utilisateurs pilotes via le côté serveur.

## Étape 5. Tests

### Tests internes
- [ ] Tests alpha passés (≥ 50 utilisateurs).
- [ ] Tous les scénarios critiques (20 tests) passés.
- [ ] Erreurs trouvées corrigées.

### Tests bêta
- [ ] Build bêta distribué via RuStore Beta ou un canal interne.
- [ ] Retours obtenus (≥ 100 utilisateurs).
- [ ] Sessions sans crash ≥ 99,8 %.
- [ ] Erreurs trouvées corrigées.

## Étape 6. Soumission à la modération

- [ ] Tous les champs remplis.
- [ ] Tous les artefacts téléchargés.
- [ ] Demande de vérification envoyée dans RuStore Console.

### Délai d'examen
**1–3 jours ouvrables** (standard).

### Issues possibles
- ✓ Approuvé — publication dans le catalogue.
- ⚠ Revoir — commentaires du modérateur → faire les corrections → resoumettre.
- ✗ Rejeté — violation critique → analyse → reprise → resoumettre.

## Étape 7. Après publication

### Surveillance
- [ ] Surveillance des téléchargements dans RuStore Console.
- [ ] Surveillance des avis utilisateurs.
- [ ] Crash reporting connecté (AppMetrica / alternative nationale).
- [ ] Surveillance des KPIs principaux (DAU, MAU, retention).

### Support
- [ ] L'équipe de support est prête à répondre aux demandes.
- [ ] E-mail support@dp-rsfsr.ru surveillé 24/7.
- [ ] Chat-bot dans l'application fonctionne.

### Mises à jour
- [ ] Calendrier des mises à jour régulières (1–2 fois par mois).
- [ ] Procédure hot-fix pour les vulnérabilités critiques.
- [ ] Compatibilité descendante de l'API ≥ 12 mois.
