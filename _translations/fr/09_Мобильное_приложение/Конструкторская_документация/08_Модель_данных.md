## 2. Modèle de données côté serveur

Le modèle complet de données est décrit dans la documentation technique de la plateforme (`01_Техническая_документация/06_DLT_реестр_ЦФА.md`). Le client utilise les projections REST du modèle côté serveur.

### Entités clés
- **Identity** (utilisateur) — lien avec ESIA.
- **KycRecord** — enregistrements de vérification KYC.
- **Asset** — actif lié à EGRN/Rospatent/dépositaire.
- **Valuation** — évaluation avec partenaire, méthode, assurance.
- **SmartContract** — contrat intelligent avec signatures de toutes les parties.
- **CFAToken** — token dans le registre DLT avec historique complet.
- **Escrow** — enregistrement d'escrow de la Banque de Russie.
- **Treasury** — enregistrements du Trésor fédéral.
- **NationalProgramme** — programme de projet national.
- **InsurancePolicy** — police d'assurance.
- **EgrnRecord** — enregistrement de la charge.
- **Notification** — notification utilisateur.
- **AuditLog** — journal immuable de toutes les opérations.

## 3. Relations entre entités

```
User ─< Asset ─< Valuation
        │
        └─< SmartContract ──> CFAToken ──> Escrow ──> Treasury
                                  │
                                  ├─> InsurancePolicy
                                  ├─> EgrnRecord
                                  └─< Transaction (versements)

User ─< Notification
User ─< AuditLog (read-only)
```

## 4. Cycle de vie des données

### 4.1. Asset

```
DRAFT → PENDING_VALUATION → VALUED → CONTRACT_PENDING →
  ACTIVE → [RETURNED | REINVESTED | EXCHANGED]
```

### 4.2. CFA

```
PENDING_ISSUE → ISSUED → IN_CB_PORTFOLIO → ACTIVE →
  [MATURED → RETURNED | EXCHANGED_MENA | INSURED_CASE | REINVESTED]
```

## 6. Sauvegarde

Les données utilisateur sont répliquées automatiquement côté serveur. L'appareil ne stocke que le cache pour le mode hors ligne. Lors du changement d'appareil, toutes les données sont restaurées après la première autorisation sur le nouvel appareil.

## 7. Suppression des données utilisateur

Lors de la suppression du compte utilisateur conformément à FZ-152 :
1. Données sur l'appareil — supprimées immédiatement.
2. Données en BD d'exploitation — supprimées sous 30 jours.
3. Données du registre DLT (pour les AFN actifs) — la charge doit être levée ; ensuite, l'enregistrement du registre reste sous forme archivée avec mention de la suppression des données personnelles du propriétaire, mais en conservant la chronologie (exigence de reporting de supervision).
4. Sauvegardes — supprimées selon le calendrier de rotation (jusqu'à 7 ans de conservation selon les exigences de la BdR).
