# Manuel d'installation et d'administration

## CPFSR — application mobile

**Identifiant :** РСФСР.МП.05.РУ
**Norme :** GOST 19.503-79

---

## 1. Informations générales sur le programme

L'application est livrée sous forme de paquets d'installation pour chaque plateforme cible. Disponible en plus en mode PWA — installation depuis le navigateur sans intermédiaire de store.

| Plateforme | Paquet | Taille |
|---|---|---|
| Android | `.apk` / `.aab` | jusqu'à 30 Mo |
| iOS | `.ipa` | jusqu'à 35 Mo |
| Windows | `.msix` / `.appx` | jusqu'à 40 Mo |
| PWA | `manifest.json` + `index.html` + `sw.js` | jusqu'à 1 Mo de téléchargement initial |

## 2. Installation du programme

### 2.1. Installation depuis RuStore (Android, recommandée pour la FR)

1. Ouvrez RuStore sur l'appareil.
2. Dans la recherche, saisissez « CPFSR » ou « Service russe de financement du développement social ».
3. Tapez « Installer ».
4. Attendez le téléchargement et l'installation (~30 Mo).
5. Ouvrez l'application via l'icône sur le bureau.

#### Exigences minimales
- Android 9 (API 28) et supérieur.
- 200 Mo de mémoire libre.

### 2.2. Installation depuis l'App Store (iOS)

1. Ouvrez l'App Store.
2. Dans la recherche, saisissez « CPFSR ».
3. Tapez « Obtenir ».
4. Confirmez avec Touch ID / Face ID / mot de passe Apple ID.
5. Attendez l'installation.

#### Exigences minimales
- iOS 15 et supérieur.
- 200 Mo de mémoire libre.

### 2.3. Installation depuis Google Play (Android)

Similaire à RuStore. Disponible sous réserve d'approbations réglementaires régionales.

### 2.4. Installation depuis Microsoft Store (Windows)

1. Ouvrez Microsoft Store.
2. Dans la recherche, saisissez « CPFSR ».
3. Tapez « Obtenir ».
4. Attendez l'installation.

#### Exigences minimales
- Windows 10 (1809) ou Windows 11.
- 500 Mo de mémoire libre.
- Architecture : x64, ARM64.

### 2.5. Installation en tant que PWA (toute plateforme)

Le mode PWA fonctionne depuis tout navigateur moderne sans intermédiaire de store.

#### Installation sur Android (Chrome, Edge) :
1. Ouvrez le site de l'application dans le navigateur.
2. Dans le menu du navigateur, sélectionnez « Installer l'application » ou une bannière d'installation apparaîtra.
3. Confirmez l'installation.
4. L'icône sera ajoutée à l'écran d'accueil.

#### Installation sur iOS (Safari) :
1. Ouvrez le site de l'application dans Safari.
2. Tapez « Partager ».
3. Choisissez « Sur l'écran d'accueil ».
4. Confirmez.

#### Installation sur Windows (Edge, Chrome) :
1. Ouvrez le site de l'application dans le navigateur.
2. Une icône d'installation apparaîtra dans la barre d'adresse.
3. Tapez-la et confirmez.

## 3. Configuration du programme

### 3.1. Premier lancement
Au premier lancement, l'application demande :
- l'autorisation pour les push-notifications ;
- l'autorisation d'utiliser la biométrie (facultatif) ;
- l'autorisation via Gosuslugi.

### 3.2. Paramètres de configuration
La plupart des paramètres de configuration sont définis automatiquement à partir du manifest de l'application. Les paramètres utilisateur sont disponibles dans la section « Profil → Paramètres » :
- Langue d'interface.
- Thème.
- Taille de police.
- Biométrie.
- Code PIN.
- Authentification à deux facteurs.
- Push-notifications.

### 3.3. Stockage des données
Les données sensibles sont stockées exclusivement dans les stockages protégés du système d'exploitation :
- iOS Keychain.
- Android Keystore.
- Windows Credential Locker.

Le cache des données de l'application est stocké sous forme chiffrée dans le répertoire de l'application. La taille du cache est limitée à 200 Mo ; en cas de dépassement, les anciennes données sont automatiquement supprimées.

## 4. Mise à jour du programme

### 4.1. Mises à jour automatiques
Via le mécanisme standard du store d'applications (RuStore, App Store, Google Play, Microsoft Store) avec notification à l'utilisateur.

### 4.2. Mises à jour forcées
Pour les vulnérabilités critiques, le mécanisme in-app force-update est appliqué : au lancement de l'application, la version minimale acceptable est vérifiée ; en cas de non-conformité, l'application bloque le fonctionnement jusqu'à la mise à jour.

### 4.3. Mises à jour PWA
Le Service Worker vérifie automatiquement les mises à jour en arrière-plan et active la nouvelle version au lancement suivant. L'utilisateur peut forcer la mise à jour via « Profil → Paramètres → Mettre à jour l'application ».

## 5. Désinstallation du programme

### 5.1. Depuis l'appareil
Désinstallation standard via l'interface du système d'exploitation :
- Android : tap long sur l'icône → « Désinstaller ».
- iOS : tap long sur l'icône → « Supprimer l'app ».
- Windows : Paramètres → Applications → CPFSR → Désinstaller.
- PWA : clic droit sur l'icône → « Désinstaller ».

### 5.2. Suppression des données
Lors de la désinstallation, sont automatiquement supprimés :
- le cache sur l'appareil ;
- les secrets enregistrés dans le stockage protégé ;
- les paramètres de l'application.

Les données utilisateur côté serveur sont supprimées uniquement sur demande officielle via la section « Profil → Sécurité → Supprimer le compte » ou via le service de support conformément aux exigences de FZ-152.

## 6. Administration

### 6.1. Distribution en environnement corporatif
Pour les utilisateurs publics et corporatifs, sont disponibles :
- Distribution MDM via le portail corporatif.
- Apple Business Manager / Google Workspace Endpoint Management.
- Intune (pour Windows).

### 6.2. Surveillance
Le côté serveur de la plateforme enregistre toutes les requêtes de l'application. Les métriques sont disponibles via le reporting réglementaire.

### 6.3. Contact du support
- Via la section « Profil → Aide et support » dans l'application.
- Par téléphone de la hotline (indiqué dans l'application).
- Via le formulaire de retour dans la grande présentation du projet.

## 7. Sécurité

### 7.1. Protection contre la compromission de l'appareil
En cas de détection de jailbreak/root, l'application :
1. Affiche un avertissement sur un environnement non sécurisé.
2. Bloque les fonctions d'émission d'AFN et de signature des contrats intelligents.
3. Conserve uniquement la lecture seule pour les AFN déjà émis.

### 7.2. Protection contre la perte de l'appareil
En cas de perte de l'appareil, l'utilisateur doit :
1. Se connecter à l'espace personnel depuis un autre appareil via PWA ou via le site web.
2. Aller dans « Profil → Paramètres → Sécurité ».
3. Taper « Bloquer toutes les sessions ».

### 7.3. Récupération d'accès
La récupération du mot de passe et de l'accès s'effectue via Gosuslugi selon la procédure standard de récupération du compte ESIA.
