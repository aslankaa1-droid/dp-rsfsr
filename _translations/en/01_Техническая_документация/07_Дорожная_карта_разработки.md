# Development roadmap of CPFSR

The document elaborates the implementation plan of the platform in line with the author's concept (term — from 18 months) and in the logic of three phases: preparation → pilot → roll-out.

---

## 1. Planning principles

1. The author's concept fixes the implementation term of the operator platform as **from 18 months**.
2. Technological development is conducted **in parallel** with the preparation of the regulatory acts package (see the Legal Foundation).
3. Minimum Viable Product (MVP) principle: the first operational version with the basic DFA life cycle is launched 12 months from the start.
4. Technology releases — under a CI/CD model with regular releases (two-week iterations during active development).

---

## 2. Phase 1. Preparation (2025–2026)

### Q1 2025 – Q2 2025 (months 1–6) — Architecture and foundation

| Milestone | Description |
|---|---|
| M1.1 | Approval of the concept with the Bank of Russia. Formation of an inter-agency working group. |
| M1.2 | Preparation of drafts of federal laws (amendments to FZ-86 and FZ-259); approval with the Ministry of Finance and the Ministry of Digital Development. |
| M1.3 | Establishment of the operator legal entity (JSC / LLC), authorised capital. |
| M1.4 | Approval of the technology stack and selection of the DLT vendor. |
| M1.5 | Selection of the pool of partners (valuation and insurance organisations). |
| M1.6 | Installation of the infrastructure in the first data centre (Astra Linux, K8s, DBMS). |
| M1.7 | Development of detailed Terms of Reference (under GOST 34.602). |

### Q3 2025 – Q4 2025 (months 7–12) — MVP development

| Milestone | Description |
|---|---|
| M2.1 | Development of core services: KYC, personal account, valuation module. |
| M2.2 | Integrations of the basic set: ESIA, EGRN, SMEV 3. |
| M2.3 | Development of the DFA DLT register. |
| M2.4 | Development of the smart contracts module. |
| M2.5 | Development of the mortgagor's web portal. |
| M2.6 | Basic information security system: perimeter, SIEM, EDR; first GOST R 57580.1 audit. |
| M2.7 | Internal pentest and remediation of findings. |
| M2.8 | Internal load tests. |
| M2.9 | Submission of documents to the Bank of Russia for inclusion in the IS operators register (779-P). |

### Q1 2026 (months 13–15) — Certification and state registration

| Milestone | Description |
|---|---|
| M3.1 | Categorisation of the IS as a CII object; notification of FSTEC. |
| M3.2 | Connection to GosSOPKA. |
| M3.3 | Certification of information security tools by FSTEC / FSB. |
| M3.4 | Inclusion in the Bank of Russia IS operators register. |
| M3.5 | Inclusion in the register of Russian software (RF Government Resolution No. 1236). |
| M3.6 | External pentest by an accredited laboratory. |
| M3.7 | Audit of compliance with GOST R 57580.2. |

### Q2 2026 (months 16–18) — Readiness for the pilot

| Milestone | Description |
|---|---|
| M4.1 | Deployment of the second data centre (geo-distribution). |
| M4.2 | Preparation of the iOS and Android mobile applications (store-ready). |
| M4.3 | Conclusion of agreements with the Federal Treasury. |
| M4.4 | Selection of the first pool of pilot users. |
| M4.5 | Approval of the target national-project programme for the pilot. |
| M4.6 | Test issue (sandbox mode without real funds). |
| **Checkpoint: launch of the operator platform (18 months)** |  |

## 3. Phase 2. Piloting and main phase (2026–2027)

### Q3 2026 – Q1 2027 (months 19–27)

| Milestone | Description |
|---|---|
| M5.1 | Launch of the pilot on a limited pool of mortgagors (up to 1,000 individuals + 50 legal entities). |
| M5.2 | First real infrastructure funding (horizon T+36 … T+60 months) against the pool of collateral DFAs. |
| M5.3 | First targeted financing of a specific national project through the Federal Treasury. |
| M5.4 | Monitoring of the operation and collection of feedback. |
| M5.5 | Deployment of the third data centre. |
| M5.6 | Release of the mobile application for Windows / desktop (Capacitor v6+ + PWA installable; MSIX via Capacitor Windows). |
| M5.7 | Expansion of the partner pool. |

### Q2 2027 – Q4 2027 (months 28–36)

| Milestone | Description |
|---|---|
| M6.1 | Removal of pilot restrictions; industrial operation for all individuals. |
| M6.2 | Expansion to legal entities — the mass segment. |
| M6.3 | Connection of state enterprises, NCOs, foundations. |
| M6.4 | Connection of all types of collateral real-estate assets. |
| M6.5 | First DFA exchange (an object is commissioned, the owner's DFA is replaced by the DFA of the new object). |
| M6.6 | Audit of the results of the first phase; public report. |
| M6.7 | Preparation for phase 3 (roll-out). |

## 6. Critical dependencies

| Dependency | Type | Mitigation |
|---|---|---|
| Adoption of amendments to FZ-86 and FZ-259 | External — legislative | Support in the State Duma and the Government; parallel development of the platform under current law |
| Obtaining a Bank of Russia licence (779-P) | External — regulatory | Early involvement of the CBR in the project work |
| Integration with the digital rouble | External — technical | Approval with the CBR digital rouble platform team |
| Availability of domestic DLT platforms | External — technological | Pilot with 2 suppliers; selection of the final vendor by PoC results |
| Recruitment of personnel with DFA expertise | Internal — HR | Training programme jointly with HSE / RANEPA |
