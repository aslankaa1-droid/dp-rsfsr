# Architecture diagrams of CPFSR

The document contains textual descriptions of the platform's C4 diagrams (Mermaid + structured description). The visual diagrams will be developed by a designer as part of the accompanying package.

---

## 1. C4 — Level 1: System Context

```mermaid
graph TB
    subgraph "Environment"
        OWN["Mortgagor<br/>individual / legal entity / state enterprise / NCO"]
        VAL["Valuation organisation<br/>(platform partner)"]
        INS["Insurance organisation<br/>(platform partner)"]
        CB["Central Bank<br/>of the Russian Federation"]
        FK["Federal Treasury"]
        FR["Rosfinmonitoring"]
        ROSR["Rosreestr (EGRN)"]
        GOSUS["Gosuslugi (ESIA / EBS)"]
        SMEV["SMEV 3"]
        NPRA["State client<br/>(ministries, agencies)"]
        FSB["FSTEC / FSB / NCCCI<br/>(CII supervision)"]
    end

    RSFSR["**CPFSR**<br/>Operator platform<br/>for the issue of DFAs"]

    OWN -->|asset submission| RSFSR
    RSFSR -->|valuation| VAL
    VAL -->|result| RSFSR
    RSFSR -->|insurance| INS
    RSFSR <-->|DFA / register| CB
    CB <-->|Targeted funding / escrow| FK
    FK -->|financing| NPRA
    RSFSR -->|FZ-115 reporting| FR
    RSFSR <-->|encumbrances| ROSR
    RSFSR <-->|identification| GOSUS
    RSFSR -->|inter-agency exchange| SMEV
    RSFSR <-->|CII supervision| FSB
```

### Description
CPFSR is the central platform connecting four types of stakeholders:
- **Asset owners** — natural and legal persons, state enterprises, NCOs, foundations.
- **Partners** — valuation and insurance organisations accredited by the platform.
- **Regulators** — Bank of Russia, Rosfinmonitoring, FSTEC/FSB.
- **State bodies** — Federal Treasury and state clients of national projects.

```mermaid
    end
```

```mermaid
    K8S2 <--> K8S3
    K8S1 <--> K8S3
    PG1 <-->|sync repl| PG2
    PG2 <-->|sync repl| PG3
    DLT1 <-->|consensus| DLT2
    DLT2 <-->|consensus| DLT3
    DLT1 <-->|consensus| DLT3
    BACK <-- K8S1
    BACK <-- K8S2
    BACK <-- K8S3
```

### Description
- Active-active-active topology across 3 data centres located in different federal districts.
- Synchronous DB replication.
- Distributed DLT-node consensus.
- A reserve "cold" data centre for DR scenarios.

## 6. Sequence — DFA issue (E2E)

```mermaid
sequenceDiagram
    participant U as Mortgagor
    participant APP as Mobile App
    participant API as API Gateway
    participant KYC as KYC Service
    participant CAT as Catalog
    participant ROSR as EGRN
    participant VAL as Valuation
    participant SC as Smart Contract Engine
    participant CFA as CFA Issuer
    participant DLT as DLT Registry
    participant CB as CBR
    participant FK as Fed. Treasury

    U->>APP: Registration
    APP->>API: AuthN (ESIA)
    API->>KYC: FZ-115 check
    KYC-->>API: OK
    U->>APP: Asset submission
    APP->>API: createAsset
    API->>CAT: createAsset
    CAT->>ROSR: getAssetInfo
    ROSR-->>CAT: EGRN data
    CAT-->>API: assetId
    U->>APP: Request valuation
    APP->>API: requestValuation
    API->>VAL: createValuationOrder
    VAL-->>API: pending
    Note over VAL: Partner valuer<br/>performs the valuation
    VAL-->>API: valuationDone
    U->>APP: Sign smart contract
    APP->>API: signContract (UKEP)
    API->>SC: build & sign
    SC->>CFA: requestIssue
    CFA->>DLT: mint(cfaId)
    DLT-->>CFA: OK
    CFA->>CB: transferToCB
    CB-->>CFA: receipt
    CB->>FK: emission
    FK-->>CB: confirmed
    CB-->>CFA: ACTIVE
    CFA-->>API: cfaId ACTIVE
    API-->>APP: success
    APP-->>U: DFA issued
```

## 7. Sequence — DFA exchange

```mermaid
sequenceDiagram
    participant FK as Fed. Treasury
    participant NPRA as National-project client
    participant ROSR as EGRN
    participant CFA as CFA Issuer
    participant DLT as DLT Registry
    participant CB as CBR
    participant OWN as Mortgagor

    NPRA->>FK: commissioning act
    FK->>ROSR: title registration
    ROSR-->>FK: certificate
    FK->>CFA: request for the issue of a new DFA
    CFA->>DLT: mint(cfaIdNew, new appraisal)
    DLT-->>CFA: OK
    CFA->>CB: request for exchange of the owner's old DFA for the new one
    CB-->>CFA: confirmed
    DLT->>DLT: status MENA_DONE
    CFA->>ROSR: remove the encumbrance on the old one
    ROSR-->>CFA: OK
    CFA->>OWN: reward
    OWN-->>CFA: ACK
```
