Installation Manual

Автор: Кагиров А.-Х.А. · Свидетельство о депонировании №4011265 от 19.12.2024 · Center Group Company · Engagement: CPFSR-RF-DESIGN-2026

Installation and administration manual

CPFSR — mobile application

Identifier: РСФСР.МП.05.РУ Standard: GOST 19.503-79

1. General information about the programme

The application is delivered in the form of installation packages for each target platform. Additionally available in PWA mode — installation from the browser without store mediation.

Platform Package Size
Android .apk / .aab up to 30 MB
iOS .ipa up to 35 MB
Windows .msix / .appx up to 40 MB
PWA manifest.json + index.html + sw.js up to 1 MB initial download

2. Programme installation

2.1. Installation from RuStore (Android, recommended for the RF)

  1. Open RuStore on the device.
  2. In the search, enter “CPFSR” or “Russian Service for Social Development Financing”.
  3. Tap “Install”.
  4. Wait for the download and installation (~30 MB).
  5. Open the application from the icon on the desktop.

Minimum requirements

  • Android 9 (API 28) and above.
  • 200 MB of free memory.

2.2. Installation from App Store (iOS)

  1. Open App Store.
  2. In the search, enter “CPFSR”.
  3. Tap “Download”.
  4. Confirm with Touch ID / Face ID / Apple ID password.
  5. Wait for the installation.

Minimum requirements

  • iOS 15 and above.
  • 200 MB of free memory.

2.3. Installation from Google Play (Android)

Similar to RuStore. Available subject to regional regulatory approvals.

2.4. Installation from Microsoft Store (Windows)

  1. Open Microsoft Store.
  2. In the search, enter “CPFSR”.
  3. Tap “Get”.
  4. Wait for the installation.

Minimum requirements

  • Windows 10 (1809) or Windows 11.
  • 500 MB of free memory.
  • Architecture: x64, ARM64.

2.5. Installation as a PWA (any platform)

The PWA mode works from any modern browser without store mediation.

Installation on Android (Chrome, Edge):

  1. Open the application site in the browser.
  2. In the browser menu, select “Install application” or an installation banner will appear.
  3. Confirm the installation.
  4. The icon will be added to the home screen.

Installation on iOS (Safari):

  1. Open the application site in Safari.
  2. Tap “Share”.
  3. Choose “To Home Screen”.
  4. Confirm.

Installation on Windows (Edge, Chrome):

  1. Open the application site in the browser.
  2. An installation icon will appear in the address bar.
  3. Tap it and confirm.

3. Programme configuration

3.1. First launch

On first launch, the application requests: - permission for push notifications; - permission to use biometrics (optional); - authorisation via Gosuslugi.

3.2. Configuration parameters

Most configuration parameters are set up automatically from the application manifest. User settings are available in the “Profile → Settings” section: - Interface language. - Theme. - Font size. - Biometrics. - PIN code. - Two-factor authentication. - Push notifications.

3.3. Data storage

Sensitive data is stored exclusively in protected storages of the operating system: - iOS Keychain. - Android Keystore. - Windows Credential Locker.

The application data cache is stored in encrypted form in the application directory. The cache size is limited to 200 MB; when exceeded, old data is automatically removed.

4. Programme update

4.1. Automatic updates

Via the standard app-store mechanism (RuStore, App Store, Google Play, Microsoft Store) with notification to the user.

4.2. Forced updates

For critical vulnerabilities, the in-app force-update mechanism is applied: on application launch, the minimum acceptable version is checked; if it does not match, the application blocks operation until the update.

4.3. PWA updates

The Service Worker automatically checks for updates in the background and activates the new version on the next launch. The user may force an update via “Profile → Settings → Update application”.

5. Programme uninstallation

5.1. From the device

Standard uninstallation through the operating system interface: - Android: long tap on the icon → “Uninstall”. - iOS: long tap on the icon → “Remove App”. - Windows: Settings → Apps → CPFSR → Uninstall. - PWA: right click on the icon → “Uninstall”.

5.2. Data deletion

On uninstallation, the following are automatically removed: - the cache on the device; - saved secrets in the protected storage; - application settings.

Server-side data of the user are deleted only upon an official request via the “Profile → Security → Delete account” section or through the support service in accordance with the requirements of FZ-152.

6. Administration

6.1. Distribution in the corporate environment

For state and corporate users, available: - MDM distribution via the corporate portal. - Apple Business Manager / Google Workspace Endpoint Management. - Intune (for Windows).

6.2. Monitoring

The server-side of the platform records all the application’s requests. Metrics are available through regulatory reporting.

6.3. Contacting support

  • Through the “Profile → Help and support” section in the application.
  • By the hotline phone (indicated in the application).
  • Through the feedback form in the project’s big presentation.

7. Security assurance

7.1. Protection against device compromise

On detection of jailbreak/root, the application: 1. Displays a warning about an unsafe environment. 2. Blocks DFA issue and smart-contract signing functions. 3. Preserves only the read-only of already issued DFAs.

7.2. Protection against device loss

On loss of the device, the user shall: 1. Sign in to the personal cabinet from another device via the PWA or the website. 2. Go to “Profile → Settings → Security”. 3. Tap “Block all sessions”.

7.3. Access recovery

Password and access recovery are carried out through Gosuslugi under the standard ESIA account recovery procedure.

The document complies with the requirements of GOST 19.503-79 “System programmer’s manual”.
