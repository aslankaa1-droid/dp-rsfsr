Promotional materials for publication in RuStore

Composition of the pack

1. Application icons

Purpose Size File Format
Icon in the catalogue 512 × 512 px icons/store-512.png PNG
Application icon (adaptive) 192 × 192 px icons/icon-192.svg SVG
Application icon (full) 512 × 512 px icons/icon-512.svg SVG

Icons are generated from the source SVG logo Приложение/icons/icon-512.svg.

2. Feature graphic / banner

Purpose Size Description
Feature graphic 1024 × 500 px Main catalogue banner; includes the logo, name, key phrase

3. Screenshots

Minimum RuStore requirements

  • A minimum of 5 screenshots, a maximum of 8.
  • Aspect ratio: 16:9 (horizontal) or 9:16 (vertical).
  • Minimum size: 320 × 480.
  • Recommended size: 1080 × 1920 (vertical).
  • Format: PNG, JPEG.

4. Promo video

Parameter Value
Duration 15–30 seconds
Format MP4 (H.264 + AAC)
Resolution 1080 × 1920 (vertical)
Audio with voice-over or subtitles

Scenario (15 seconds): - 0–3 s: logo + name. - 3–6 s: home — balance and assets. - 6–9 s: asset submission. - 9–12 s: smart contract signing. - 12–15 s: “DFA issued” + slogan.

Instructions for screenshot preparation

  1. Open the Application in a browser in phone simulation mode (DevTools → Mobile, resolution 1080 × 1920).
  2. Navigate to the relevant screen.
  3. Take a screenshot via DevTools (Cmd/Ctrl + Shift + P → “Capture screenshot”).
  4. If necessary — add captions and styling in a graphics editor.

Icon generation command (requires ImageMagick or Pillow)

# Using Pillow / Python
python -c "
from PIL import Image
import cairosvg, io

for size in [44, 71, 96, 150, 192, 310, 512]:
    cairosvg.svg2png(url='Приложение/icons/icon-512.svg', write_to=f'store-{size}.png', output_width=size, output_height=size)
"

Alternatively — @capacitor/assets CLI automatically generates all sizes.
