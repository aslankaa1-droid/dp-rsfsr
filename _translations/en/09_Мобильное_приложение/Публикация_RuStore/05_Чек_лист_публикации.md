# Publication checklist for RuStore

## CPFSR — mobile application

---

## Stage 1. Developer preparation

- [ ] Account registered in RuStore Console: https://console.rustore.ru
- [ ] Account confirmed via Gosuslugi (for Russian developers).
- [ ] Documents uploaded:
  - [ ] passport or other document for an individual;
  - [ ] founding documents for a legal entity;
  - [ ] INN certificate.
- [ ] RuStore developer agreement signed.

## Stage 2. Application preparation

### Technical artefacts
- [ ] AAB file built (`app-release.aab`).
- [ ] AAB size ≤ 150 MB (RuStore limit).
- [ ] AAB signed with a release keystore (not a debug keystore).
- [ ] versionCode and versionName aligned.
- [ ] minSdkVersion = 28 (Android 9).
- [ ] targetSdkVersion = 34 (Android 14).
- [ ] applicationId = `ru.rsfsr.app`.

### Compliance
- [ ] The application works on the minimum target Android version (Android 9).
- [ ] The application has been tested on at least 5 different devices.
- [ ] Crash-rate < 0.5% per internal test data.

### Content
- [ ] 512 × 512 PNG icon.
- [ ] 1024 × 500 PNG feature graphic.
- [ ] A minimum of 5 screenshots (1080 × 1920 recommended).
- [ ] Promo video (optional, but recommended).
- [ ] Brief description (up to 80 characters).
- [ ] Full description (up to 4,000 characters).
- [ ] "What's new" text (up to 500 characters).
- [ ] Keywords.

### Legal documents
- [ ] Privacy Policy published at `https://dp-rsfsr.ru/privacy` (or a temporary domain).
- [ ] Terms of Use published at `https://dp-rsfsr.ru/terms`.
- [ ] Developer website address confirmed.
- [ ] Support contact email active.

## Stage 3. Filling in the RuStore Console card

### Main information
- [ ] Name: "CPFSR".
- [ ] Subtitle: "Russian Service for Social Development Financing".
- [ ] Category: Finance.
- [ ] Subcategory: Investment.
- [ ] Age category: 18+.
- [ ] Price: Free.

### Description
- [ ] Brief description filled in.
- [ ] Full description filled in.
- [ ] Keywords specified.
- [ ] "What's new" filled in.

### Graphics
- [ ] Icon uploaded.
- [ ] Feature graphic uploaded.
- [ ] 5–8 screenshots uploaded.
- [ ] Promo video uploaded (if available).

### Technical information
- [ ] AAB file uploaded.
- [ ] Mapping (mapping.txt for ProGuard) uploaded.
- [ ] Signatures in order (debug keystore not used).

### Content rating
- [ ] IARC questionnaire / Russian labelling system filled in.
- [ ] Age category 18+ set.

### Localisation
- [ ] Russian language — primary, filled in completely.
- [ ] English language — filled in.
- [ ] French language — filled in (optional).
- [ ] Arabic language — filled in (optional).

### Legal information
- [ ] Link to the Privacy Policy.
- [ ] Link to the Terms of Use.
- [ ] Developer contact information.
- [ ] Stated that the application does not use advertising.
- [ ] Stated that the application has no in-app purchases.

### Financial information (where applicable)
- [ ] Type of financial activity specified.
- [ ] Copies of licences uploaded (Bank of Russia — after obtainment).
- [ ] Link to state systems (Gosuslugi, EGRN) specified.

## Stage 4. Security and regulatory matters

### Before production publication
- [ ] FSTEC of Russia certificate obtained (K1 / CII-1).
- [ ] FSB of Russia certificate obtained (cryptographic information security tool).
- [ ] Roskomnadzor notification of inclusion in the personal data operators register obtained.
- [ ] Application included in the Unified Register of Russian Software (RF Government Resolution No. 1236).
- [ ] Operator included in the Bank of Russia IS operators register (Regulation No. 779-P).

### Alternatively: for pilot mode
- [ ] Bank of Russia permission for pilot operation obtained.
- [ ] The application explicitly indicates "Pilot mode" status (in the About section).
- [ ] Access is restricted to the pilot user pool via the server-side.

## Stage 5. Testing

### Internal testing
- [ ] Alpha testing passed (≥ 50 users).
- [ ] All critical scenarios (20 tests) passed.
- [ ] Found errors remediated.

### Beta testing
- [ ] Beta build distributed via RuStore Beta or an internal channel.
- [ ] Feedback received (≥ 100 users).
- [ ] Crash-free sessions ≥ 99.8%.
- [ ] Found errors remediated.

## Stage 6. Submission for moderation

- [ ] All fields filled in.
- [ ] All artefacts uploaded.
- [ ] Review request sent in RuStore Console.

### Review period
**1–3 business days** (standard).

### Possible outcomes
- ✓ Approved — publication in the catalogue.
- ⚠ Revise — moderator comments → make corrections → resubmit.
- ✗ Rejected — critical violation → analysis → rework → resubmit.

## Stage 7. After publication

### Monitoring
- [ ] Monitoring of downloads in RuStore Console.
- [ ] Monitoring of user reviews.
- [ ] Crash reporting connected (AppMetrica / a domestic alternative).
- [ ] Monitoring of key KPIs (DAU, MAU, retention).

### Support
- [ ] The support team is ready to respond to requests.
- [ ] Email support@dp-rsfsr.ru monitored 24/7.
- [ ] Chat-bot in the application works.

### Updates
- [ ] Schedule of regular updates (1–2 times per month).
- [ ] Hot-fix procedure for critical vulnerabilities.
- [ ] API backward compatibility ≥ 12 months.
