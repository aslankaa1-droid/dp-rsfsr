# Specification of the CPFSR mobile application

**Platforms:** iOS / Android / Windows.
**Target audience:** mortgagors — natural and legal persons, representatives of state enterprises, sole traders, NCOs and foundations.
**Key objectives:** to provide the secure remote management of assets, the issue and servicing of DFAs, the receipt of remuneration.

---

## 1. Technology stack

### 1.1. Cross-platform implementation
- **Base platform:** PWA (HTML/CSS/JS + TypeScript), Service Worker for offline mode and background updating, manifest with shortcuts and share_target.
- **iOS / Android wrapper:** Capacitor v6+ (native iOS/Android wrappers over the PWA core); a single code base, native access to biometrics, GOST cryptographic information security tools, Secure Enclave/Strongbox, certificate pinning, push, share_target.
- **Windows desktop:** PWA via Edge WebView2 (a separate installation via Microsoft Store / MSIX) or edge_side_panel.
- **Common business logic:** TypeScript + WebAssembly modules for critical functions (cryptography, signing).
- **Design system:** based on the project brand tokens (`_shared/brand.css`), with native overrides for iOS (SF Pro) and Android (Roboto).

### 1.2. Client application architecture
- Pattern: PWA Shell + lazy-loaded screens.
- State management: an internal state store in `app.js` (vanilla JS) or a Zustand adapter as complexity grows.
- Navigation: PWA back-stack via History API + drawer-aside in full-size modes.
- API client: auto-generation from OpenAPI 3.1.

### 1.3. On-device data storage
- Sensitive data: iOS Keychain / Android Keystore / Windows Credential Locker — only encrypted tokens.
- Biometric data are not stored locally — only identifiers for verification through EBS.
- Local cache — SQLCipher (encrypted SQLite).

### 1.4. Cryptography
- UKEP on the mobile device — through KriptoPro EP (mobile SDK).
- Hashing — GOST R 34.11-2012.
- TLS channels — using GOST certificates.

## 2. Security

### 2.1. Identification and authentication
1. **Primary authorisation** — through ESIA (OpenID Connect) with a confirmed account.
2. **Additional factors:**
   - Device biometrics (TouchID / FaceID / Windows Hello) for access to the application.
   - Biometrics via EBS (Bank of Russia Regulation No. 482-P) for the signing of critical operations.
   - OTP (SMS / push) for critical operations (smart contract signing).

### 2.2. Application protection
- **Anti-tampering:** Mobile Application Shielding (MAS) — domestic-class solutions.
- **Jailbreak / Root Detection** — refusal to operate on compromised devices.
- **Anti-debug** — obfuscation, protection against runtime injections.
- **Anti-screenshot** for critical screens (smart contract, OTP).
- **Certificate pinning** on TLS channels.

### 2.3. Anti-fraud protection
- Behavioural analytics (behavioural biometrics).
- Device fingerprinting.
- Geolocation and time patterns.

### 2.4. Storage of secrets
- Only in a TEE (Trusted Execution Environment) / Secure Enclave.
- No keys in the application's shared memory.

### 2.5. Compliance
- FZ-152 (personal data protection).
- FZ-187 (CII) — the application is part of a significant CII object of category 1.
- Bank of Russia Instruction No. 6635-U (IS incidents).
- OWASP MASVS Level 2+ standard.
- WCAG 2.2 AA compliance (accessibility).

## 3. Functional modules

### 3.1. Registration and identification
- Authorisation through ESIA.
- Biometric verification through EBS.
- Device binding.
- Consents to the processing of personal data.

### 3.2. Personal cabinet
- Dashboard: total balance of assets in operation, current income, DFA portfolio.
- List of assets with filters and search.
- Each asset: characteristics, history, current DFA.

### 3.3. Submission of an asset as collateral
- Selection of the property type.
- Loading data from state registers (EGRN, Rospatent, depositaries).
- Attachment of documents.
- Request for a valuation.

### 3.4. Valuation
- Viewing the status of the request.
- Valuation result with the possibility to appeal.
- Information on insurance.

### 3.5. Smart contract
- Conditions designer.
- Viewing the full text of the contract.
- Electronic signing with UKEP (mobile certificate).

### 3.6. DFA management
- List of active DFAs.
- State of each DFA in the DLT register.
- Expected events (exchange, insurance events, term expiry).

### 3.7. Income
- Payment history.
- Summary by quarter and by year.
- Download of the 2-NDFL statement.

### 3.8. Notifications
- Push notifications on critical events.
- Notifications via Gosuslugi (for consents, changes).

### 3.9. Profile
- Management of contact details.
- Management of security (biometrics, OTP channel).
- Management of consents.
- Payout details.

### 3.10. Help and support
- FAQ.
- Chat-bot.
- Connection to the support operator.

## 5. Release plan

### 5.1. iOS / Android
- Beta release via TestFlight / Google Play Internal Testing — month 10.
- Closed testing — month 12.
- Open launch — month 18 (simultaneously with the platform).
- Placement in RuStore (Ministry of Digital Development).

### 5.2. Windows / Desktop (Capacitor v6+ for desktop / PWA installable)
- Beta release — month 14 (PWA via Edge / Chrome "Install app"; a full-fledged Capacitor Windows bundle — month 16).
- Open launch — month 18.
- Placement in Microsoft Store (via the MSIX package generated by Capacitor) + alternative channels (web-distribution PWA).
- Advantage of a single PWA code base: the same sources as for iOS/Android — no parallel .NET MAUI stack.

### 5.3. Updates
- Auto-update through the standard stores.
- Forced updates for critical vulnerabilities through the in-app mechanism.

## 6. Testing

### 6.1. Types of testing
- Unit tests (Vitest + @testing-library/dom for PWA logic, JUnit/XCTest for native Capacitor plugins) — coverage ≥ 80%.
- Integration tests (Playwright Component Testing) — critical scenarios.
- E2E tests (Playwright cross-browser + Capacitor + Appium for native wrappers) — the main user flows for iOS / Android / desktop.
- Performance tests (Lighthouse CI ≥ 90 across all 4 PWA metrics) on various devices.
- Security tests (OWASP MASVS L2+ FT-021..025, ZAP / Burp, OWASP Mobile Top 10, FSTEC-certified scanners).
- Accessibility tests (axe-core, WCAG 2.2 AA, iOS VoiceOver / Android TalkBack screen readers).

### 6.2. Device pool
- iOS: iPhone 13 / 14 / 15 / 16; iPad 9-gen / Pro.
- Android: Samsung Galaxy S22+, Xiaomi Redmi Note 12, Realme C55 (budget segment).
- Windows: Surface Pro 9, typical workstations.

### 6.3. CI/CD
- Build in a secured environment with UKEP signing.
- Automatic testing on every merge.
- Release branches + tags.
