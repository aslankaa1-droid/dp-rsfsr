# Архитектурные диаграммы ЦП РСФСР

Документ содержит описания С4-диаграмм платформы в текстовом виде (Mermaid + структурированное описание). Визуальные диаграммы будут разработаны дизайнером в составе сопровождающего пакета.

---

## 1. C4 — Level 1: System Context

```mermaid
graph TB
    subgraph "Окружение"
        OWN["Залогодатель<br/>физлицо / юрлицо / госпредприятие / НКО"]
        VAL["Оценочная организация<br/>(партнёр платформы)"]
        INS["Страховая организация<br/>(партнёр платформы)"]
        CB["Центральный банк<br/>Российской Федерации"]
        FK["Федеральное казначейство"]
        FR["Росфинмониторинг"]
        ROSR["Росреестр (ЕГРН)"]
        GOSUS["Госуслуги (ЕСИА / ЕБС)"]
        SMEV["СМЭВ 3"]
        NPRA["Государственный заказчик<br/>(министерства, ведомства)"]
        FSB["ФСТЭК / ФСБ / НКЦКИ<br/>(надзор за КИИ)"]
    end

    RSFSR["**ЦП РСФСР**<br/>Платформа-оператор<br/>выпуска ЦФА"]

    OWN -->|подача актива| RSFSR
    RSFSR -->|оценка| VAL
    VAL -->|результат| RSFSR
    RSFSR -->|страхование| INS
    RSFSR <-->|ЦФА / реестр| CB
    CB <-->|Целевое фондирование / эскроу| FK
    FK -->|финансирование| NPRA
    RSFSR -->|отчётность 115-ФЗ| FR
    RSFSR <-->|обременения| ROSR
    RSFSR <-->|идентификация| GOSUS
    RSFSR -->|межведомственный обмен| SMEV
    RSFSR <-->|надзор КИИ| FSB
```

### Описание
ЦП РСФСР — центральная платформа, соединяющая четыре типа стейкхолдеров:
- **Собственники активов** — физические и юридические лица, государственные предприятия, НКО, фонды.
- **Партнёры** — оценочные и страховые организации, аккредитованные платформой.
- **Регуляторы** — Банк России, Росфинмониторинг, ФСТЭК/ФСБ.
- **Государственные органы** — Федеральное казначейство и государственные заказчики нацпроектов.

### Описание
Платформа состоит из трёх основных слоёв:
1. **Каналы** — веб и мобильные приложения залогодателя, плюс кабинеты партнёров.
2. **Edge + сервисы** — WAF, API Gateway, микросервисы.
3. **Данные + интеграции** — БД, объектное хранилище, DLT-реестр, кеш, брокер сообщений, внешние ГИС.

```mermaid
    K8S1 <--> K8S2

    DLT1 <-->|consensus| DLT3

    BACK <-- K8S2
    BACK <-- K8S3
```

### Описание
- Active-active-active топология по 3 ЦОД, расположенных в разных федеральных округах.
- Синхронная репликация баз данных.
- Распределённый консенсус DLT-узлов.
- Резервный «холодный» ЦОД для DR-сценариев.

## 6. Sequence — выпуск ЦФА (E2E)

```mermaid
sequenceDiagram
    participant U as Залогодатель
    participant APP as Mobile App
    participant API as API Gateway
    participant KYC as KYC Service
    participant CAT as Catalog
    participant ROSR as ЕГРН
    participant VAL as Valuation
    participant SC as Smart Contract Engine
    participant CFA as CFA Issuer
    participant DLT as DLT Registry
    participant CB as ЦБ РФ
    participant FK as Фед. казначейство

    U->>APP: Регистрация
    APP->>API: AuthN (ЕСИА)
    API->>KYC: проверка 115-ФЗ
    KYC-->>API: OK
    U->>APP: Подача актива
    APP->>API: createAsset
    API->>CAT: createAsset
    CAT->>ROSR: getAssetInfo
    ROSR-->>CAT: данные ЕГРН
    CAT-->>API: assetId
    U->>APP: Запросить оценку
    APP->>API: requestValuation
    API->>VAL: createValuationOrder
    VAL-->>API: pending
    Note over VAL: Оценщик-партнёр<br/>выполняет оценку
    VAL-->>API: valuationDone
    U->>APP: Подписать смарт-контракт
    APP->>API: signContract (УКЭП)
    API->>SC: build & sign
    SC->>CFA: requestIssue
    CFA->>DLT: mint(cfaId)
    DLT-->>CFA: OK
    CFA->>CB: transferToCB
    CB-->>CFA: receipt
    CB->>FK: emission
    FK-->>CB: confirmed
    CB-->>CFA: ACTIVE
    CFA-->>API: cfaId ACTIVE
    API-->>APP: success
    APP-->>U: ЦФА выпущен
```

## 7. Sequence — мена ЦФА

```mermaid
sequenceDiagram
    participant FK as Фед. казначейство
    participant NPRA as Заказчик нацпроекта
    participant ROSR as ЕГРН
    participant CFA as CFA Issuer
    participant DLT as DLT Registry
    participant CB as ЦБ РФ
    participant OWN as Залогодатель

    NPRA->>FK: акт ввода объекта
    FK->>ROSR: регистрация ПС
    ROSR-->>FK: свидетельство
    FK->>CFA: запрос на выпуск нового ЦФА
    CFA->>DLT: mint(cfaIdNew, оценка_новая)
    DLT-->>CFA: OK
    CFA->>CB: запрос на мену старого ЦФА собственника на новый
    CB-->>CFA: подтверждено
    DLT->>DLT: status MENA_DONE
    CFA->>ROSR: снять обременение по старому
    ROSR-->>CFA: OK
    CFA->>OWN: вознаграждение
    OWN-->>CFA: ACK
```

## 8. Замечания

- Все диаграммы — концептуальные, на стадии MVP подлежат детализации.
- Визуальные C4-диаграммы будут отрисованы дизайнером по бренд-буку платформы.
- Возможна детализация Mermaid → PlantUML / ArchiMate в зависимости от инструментария команды.
