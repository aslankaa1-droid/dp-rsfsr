# Функциональные требования (PRD) ЦП РСФСР

Документ описывает функциональные модули платформы и пользовательские сценарии в формате User Story + критерии приёмки.

---

## 2. Ключевые пользовательские сценарии

### US-01. Регистрация залогодателя
**Как** собственник имущества **я хочу** зарегистрироваться на платформе через Госуслуги **чтобы** получить доступ к подаче активов на залог.

Критерии приёмки:
1. Авторизация через ЕСИА с подтверждённой учётной записью.
2. Проверка по реестру дисквалифицированных лиц и санкционным спискам (115-ФЗ).
3. Проверка ПДн на корректность.
4. При успехе — выдача персонального ID платформы (`usr-rsfsr-{n}`).
5. При несоответствии — мотивированный отказ с указанием причины.

### US-02. Подача актива на залог
**Как** залогодатель **я хочу** подать актив с автоматической подгрузкой характеристик из ЕГРН/Роспатента **чтобы** не вводить данные вручную.

Критерии приёмки:
1. Тип имущества выбран из справочника (недвижимость, транспорт, ИС, ценные бумаги).
2. Для недвижимости автозагрузка из ЕГРН по кадастровому номеру.
3. Для транспорта — из ГИС ГИБДД (если применимо).
4. Для ИС — из Роспатента.
5. Для ценных бумаг — из реестра депозитария.
6. Возможность дозаполнения вручную.
7. Прикрепление документов (свидетельство, договоры).
8. Запрос на оценку оценочной организации-партнёру.

### US-03. Оценка актива
**Как** оценщик-партнёр **я хочу** получить заявку на оценку, выполнить её и зафиксировать результат **чтобы** запустить выпуск ЦФА.

Критерии приёмки:
1. Получение оценщиком заявки в кабинете партнёра.
2. Проведение оценки согласно методике (135-ФЗ).
3. Фиксация результата в форме с обязательными полями: оценочная стоимость, метод, дата, страхование.
4. Электронная подпись результата оценщиком (УКЭП).
5. Автоматическая публикация результата в кабинете залогодателя.
6. Срок действия оценки — 6 месяцев (стандартно).

### US-04. Подписание смарт-контракта
**Как** залогодатель **я хочу** настроить условия и подписать смарт-контракт ЦФА **чтобы** запустить выпуск актива.

Критерии приёмки:
1. Конструктор условий: срок (12/24/36/60 мес.), доля номинала (по умолчанию 50% оценки), ставка вознаграждения, целевое использование.
2. Отображение полного текста смарт-контракта (PDF-предпросмотр).
3. Двухфакторное подтверждение (ЕСИА + SMS/биометрия).
4. Подписание УКЭП ГОСТ Р 34.10-2012.
5. После подписания — автоматическая регистрация обременения в ЕГРН.
6. Уведомление залогодателя о завершении.

### US-05. Выпуск ЦФА
**Как** оператор платформы **я хочу** автоматически выпустить ЦФА после подписания смарт-контракта и обременения **чтобы** передать его в Банк России.

Критерии приёмки:
1. Создание записи в DLT-реестре с уникальным ID (`cfa-{hash}-rsfsr-{year}-{seq}`).
2. Хеширование смарт-контракта ГОСТ Р 34.11-2012.
3. Уведомление Банка России через защищённый канал.
4. Подтверждение от Банка России о приёме ЦФА.
5. Возврат подтверждения залогодателю.

### US-06. Мена ЦФА
**Как** оператор **я хочу** автоматически провести мену ЦФА после ввода нового объекта в эксплуатацию **чтобы** заместить первоначальный залог собственника.

Критерии приёмки:
1. Получение свидетельства о собственности на новый объект через ЕГРН.
2. Выпуск нового ЦФА с большей оценкой.
3. Запрос на мену в Банк России.
4. Подтверждение мены, подписание Акта о взаиморасчётах всеми сторонами.
5. Снятие обременения в ЕГРН.
6. Выплата вознаграждения собственнику на банковский счёт.

### US-07. Страховой случай
**Как** оператор **я хочу** инициировать страховой ЦФА от страховой компании-партнёра **чтобы** компенсировать первоначальный ЦФА собственника.

Критерии приёмки:
1. Регистрация страхового случая с документальным подтверждением.
2. Выпуск страховой компанией собственного ЦФА через платформу.
3. Мена с первоначальным ЦФА собственника.
4. Подписание акта.
5. Выплата компенсации.

### US-08. Доходы и история
**Как** залогодатель **я хочу** видеть историю своих ЦФА и доходов **чтобы** управлять портфелем.

Критерии приёмки:
1. Дашборд с активами, статусами, текущими доходами.
2. История выплат.
3. Экспорт в PDF и Excel.
4. Налоговая справка 2-НДФЛ автоматически (для физлиц).

### US-09. Регуляторная отчётность
**Как** оператор **я хочу** формировать обязательную отчётность для регуляторов **чтобы** соответствовать требованиям 259-ФЗ и инструкциям ЦБ.

Критерии приёмки:
1. Формирование отчётности по 115-ФЗ для Росфинмониторинга.
2. Регулярные отчёты в ЦБ РФ (по 779-П).
3. Налоговая отчётность по операциям с ЦФА.
4. Согласование подписей УКЭП.

### US-10. Управление инцидентами ИБ
**Как** офицер ИБ **я хочу** получать события из SIEM **чтобы** реагировать на инциденты согласно Указанию №6635-У.

Критерии приёмки:
1. Категоризация инцидентов.
2. Уведомление Банка России в установленные сроки.
3. Уведомление ГосСОПКА.
4. Журнал реагирования.

## 3. Бизнес-правила

1. **Залоговая стоимость** = не более 50% оценочной стоимости (консервативное правило для пилотной фазы).
2. **Минимальный срок** ЦФА = 12 месяцев. Максимальный = 60 месяцев.
3. **Ставка вознаграждения** залогодателю = от 3% до 6% годовых (определяется условиями нацпроекта).
4. **Одно физлицо** может одновременно иметь не более 5 активных ЦФА.
5. **Одно юрлицо** — не более 50 активных ЦФА.
6. **Страхование** залогового актива и самого ЦФА обязательно для всех ЦФА сверх 1 млн ₽.
7. **Запрет на двойной залог:** один актив не может быть одновременно в залоге у ЦП РСФСР и иной организации.
8. **Запрет на оборот ЦФА** Банком России (по концепции автора, п. 10 пошагового процесса).

## 4. Состояния ЦФА

```
DRAFT → PENDING_VALUATION → VALUED → CONTRACT_SIGNED →
ISSUED → TRANSFERRED_TO_CB → ACTIVE →
[REINVESTED | RETURNED | INSURED_CASE | MENA_DONE]
```

Каждый переход — событие в DLT-реестре, неизменяемое.
