# Развёртывание проекта ЦП РСФСР в интернете

Проект построен исключительно из статических файлов (HTML/CSS/JS) с возможностью оффлайн-работы через Service Worker. Для размещения в интернете подойдёт **любой статический хостинг**.

---

## 1. Структура для размещения

В корне проекта (`ЦП РСФСР/`):
- `index.html` — главная точка входа (landing).
- `robots.txt` — правила индексации для поисковых систем.
- `sitemap.xml` — карта сайта для поисковых систем.
- `_shared/` — общая дизайн-система (CSS, JS, темплейты).
- `_Мастер/` — служебная папка (можно исключить из деплоя).
- `01_…` — `13_…` — папки артефактов.

**Размер пакета:** ~9,5 МБ.

---

## 2. Поддерживаемые хостинги

### A. GitHub Pages (бесплатно)

```bash
git init
git remote add origin https://github.com/<USERNAME>/rsfsr.git
git add .
git commit -m "Initial deploy"
git push -u origin main

# В Settings → Pages → Source: main / root
```

Сайт будет доступен по адресу `https://<username>.github.io/rsfsr/`.

### B. Cloudflare Pages (бесплатно, рекомендовано)

1. Подключите Git-репозиторий или загрузите ZIP через web-интерфейс.
2. Build command: оставить пустым.
3. Output directory: `/`.

Сайт получит автоматический HTTPS, CDN, аналитику.

### C. Netlify (бесплатно)

```bash
# Установить netlify-cli
npm install -g netlify-cli

# Деплой
netlify deploy --prod
```

### D. Yandex Cloud Object Storage (РФ)

1. Создайте bucket с типом «публичный».
2. Включите статический веб-сайт.
3. Установите `index.html` как индексный документ.
4. Загрузите все файлы (через консоль или `s3cmd`).

### E. VK Cloud / SberCloud / Beget

Аналогично Yandex Cloud — статический сайт на отечественной инфраструктуре.

### F. Свой сервер (Nginx)

```nginx
server {
    listen 443 ssl http2;
    server_name dp-rsfsr.ru;

    root /var/www/rsfsr;
    index index.html;

    # Кодировка для путей с кириллицей
    charset utf-8;

    # SSL
    ssl_certificate /etc/letsencrypt/live/dp-rsfsr.ru/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/dp-rsfsr.ru/privkey.pem;

    # Защита служебных папок
    location /_Мастер/ { return 404; }

    # Service Worker scope
    location /09_Мобильное_приложение/Приложение/sw.js {
        add_header Service-Worker-Allowed "/09_Мобильное_приложение/Приложение/";
        try_files $uri =404;
    }

    # Manifest правильный MIME
    location ~ \.webmanifest$  { add_header Content-Type "application/manifest+json"; }

    # Кеширование статики
    location ~* \.(css|js|svg|woff2?)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # Default
    location / { try_files $uri $uri/ =404; }

    # Безопасность
    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";
    add_header Referrer-Policy "strict-origin-when-cross-origin";
    add_header Strict-Transport-Security "max-age=63072000" always;
}
```

## 3. Доменное имя

### Рекомендуемые варианты
1. **dp-rsfsr.ru** — после регистрации оператора как государственного значения.
2. **rsfsr.ru** — через REG.RU или аналогичный регистратор.
3. **rsfsr.online**, **rsfsr.app**, **rsfsr.io** — для тестового развёртывания.

### DNS-настройки (пример Cloudflare)
```
A    @     <IP>
A    www   <IP>
CNAME app  dp-rsfsr.ru
```

## 5. Особенности для русскоязычных путей

Все файлы и папки в проекте имеют кириллические имена. Проверьте:

1. **Веб-сервер** должен корректно обрабатывать UTF-8 пути:
   - Nginx: `charset utf-8;`
   - Apache: `AddDefaultCharset UTF-8`.
2. **Браузер** автоматически URL-кодирует кириллицу в адресной строке.
3. **Sitemap.xml** содержит URL-encoded пути (так в нём уже).

Альтернативный вариант — переименовать папки в латиницу для упрощения деплоя:

```bash
# Пример скрипта переименования
mv "01_Техническая_документация" "01-tech-docs"
mv "02_Юридический_фундамент"   "02-legal"
# и т.д.
```

При этом потребуется массовое обновление путей в HTML-файлах. На данном этапе оставлены кириллические имена для удобства работы с документацией.

## 6. PWA-функциональность

Мобильное приложение в формате PWA устанавливается из браузера. Для корректной работы необходимы:

1. **HTTPS** — обязателен для Service Worker.
2. **Корректный MIME для `manifest.json`**: `application/manifest+json`.
3. **Service Worker scope** — настроен в `manifest.json` (`./`).

После деплоя пользователи смогут установить приложение через диалог «Добавить на главный экран» на iOS и «Установить приложение» в Chrome/Edge на десктопе.

## 7. Аналитика

Подключите счётчики:
- **Yandex.Metrika** — для российских пользователей.
- **AppMetrica** — для мобильного приложения.
- **Plausible / Umami** — приватная аналитика без cookies.

**НЕ подключайте Google Analytics** — это противоречит требованиям 152-ФЗ к локализации обработки ПДн.

## 8. Проверка перед деплоем

- [ ] Все четыре темы переключаются.

- [ ] Хаб документов работает; документы открываются в модальном окне.
