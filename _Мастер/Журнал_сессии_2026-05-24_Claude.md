## 5. Попытка с параллельными агентами и провал

Запустил 5 фоновых агентов (general-purpose) с детальными промтами:
- Agent Tech (диаграммы, sed B1 в 6 техфайлах, ПМИ, sitemap, sw.js).
- Agent Legal (Политика, EULA, пакет договоров).
- Agent Fin (Финмодель пересчёт, i18n.js, MC в коде).
- Agent Strat (wedge-rewrite 9 ключевых документов).
- Agent Design (PWA эмодзи→SVG, semantic, токены, RTL, Pitch_deck @media).

**Все 5 завершились с одним диагнозом:** `Edit/Write Permission denied` для подагентов в текущей sandbox-сессии. Read/Grep/Glob у них работали, запись — нет. Agent Tech / Agent Legal сами это диагностировали и попросили разрешений; Agent Fin / Agent Strat / Agent Design были остановлены вручную после получения этой же диагностики.

**Урок (сохранено в память):** в текущей конфигурации Claude Code на aslankaa подагенты (subagent_type=general-purpose) не имеют Edit/Write permission по умолчанию, даже когда у parent-сессии полные права на Edit/Write. Параллельная диспетчеризация для file-edit задач непродуктивна — нужно или менять `.claude/settings.local.json`, или делать всю работу parent-сессии напрямую.
